Start with Strait Cloud unless you have a specific reason to run the control plane yourself. Cloud gives you the shortest path to a production job: Strait runs the API, scheduler, worker plane, storage, upgrades, and billing controls. Your job code still runs in your infrastructure.

Use the community self-hosted edition when you need local evaluation, air-gapped deployment, strict data residency, or direct operational control over PostgreSQL, Redis, Sequin, and the Strait process.

| Question | Cloud | Community self-host |
|---|---|---|
| Who operates the control plane? | Strait | Your team |
| What base URL do clients use? | `https://api.strait.dev` | Your deployed URL, or `http://localhost:8080` locally |
| Where does job code run? | Your infrastructure | Your infrastructure |
| How do users create projects and API keys? | Dashboard at `app.strait.dev` | Dashboard or internal-secret API calls |
| How are upgrades applied? | Strait rolls them out | You pull, test, and deploy |
| What is the cost model? | Free tier plus paid plans | Free software, your infrastructure cost |
| Where do run records live? | Strait Cloud | Your PostgreSQL database |

## Cloud Setup

1. Open [app.strait.dev](https://app.strait.dev).
2. Create an organization and project.
3. Create an API key from `Settings > API Keys`.
4. Copy the project ID from the project settings page.
5. Export these variables in your terminal:

```bash
export STRAIT_BASE_URL=https://api.strait.dev
export STRAIT_API_KEY=strait_replace_with_your_key
export STRAIT_PROJECT_ID=proj_replace_with_your_project_id
```

Use these variables in the [Quick Start](/quickstart).

## Community Local Setup

Use this path when you want a local Strait instance before using Cloud or before planning a self-hosted deployment.

```bash
git clone https://github.com/strait-dev/strait.git
cd strait
make dev
make dev-status
```

Export the local API settings:

```bash
export STRAIT_BASE_URL=http://localhost:8080
export STRAIT_API_KEY=my-dev-secret-change-in-production-min-32-chars
export INTERNAL_SECRET=$STRAIT_API_KEY
```

Create a project:

```bash
curl -sS -X POST "$STRAIT_BASE_URL/v1/projects" \
  -H "Authorization: Bearer $INTERNAL_SECRET" \
  -H "Content-Type: application/json" \
  -d '{ "name": "Local Project", "slug": "local-project" }'
```

Export the returned `id` as `STRAIT_PROJECT_ID`, then use the same API commands from the [Quick Start](/quickstart).

## Production Self-Hosting

For a production community deployment, read [Self-Hosting](/guides/self-hosting) before exposing the API. Configure PostgreSQL backups, Redis persistence, Sequin, TLS, `INTERNAL_SECRET`, `JWT_SIGNING_KEY`, `SECRET_ENCRYPTION_KEY`, retention, monitoring, and upgrade procedures before processing customer workloads.
