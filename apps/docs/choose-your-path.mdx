Strait runs as a hosted service at [app.strait.dev](https://app.strait.dev) or as a self-hosted binary on your own infrastructure. The API, SDKs, and feature set are the same on both.

| | Cloud | Self-host |
|---|---|---|
| Setup | Sign up at app.strait.dev | Docker Compose or your own service host |
| Infrastructure | Hosted by Strait | You run PostgreSQL and Redis |
| Updates | Rolled out by Strait | You pull and deploy |
| Pricing | Free tier plus paid plans | Free, no limits |
| Data | Hosted by Strait | Your servers, your data |
| Support | Email plus SLA on paid plans | GitHub Issues |

Cloud is the right choice if you want to start in minutes and don't need to keep run data inside your own perimeter. Self-host is the right choice if you have compliance, residency, or air-gap requirements — or if you simply want full control over the stack.

## Cloud setup

Sign up at [app.strait.dev](https://app.strait.dev), create an API key under **Settings → API Keys**, install an SDK, and trigger your first job:

```typescript
import { createClient } from "@strait/ts";

const client = createClient({
  baseUrl: "https://api.strait.dev",
  auth: { type: "bearer", token: "strait_..." },
});

const run = await client.triggerJob({
  pathParams: { jobID: "my-first-job" },
  body: { payload: { message: "Hello from Strait!" } },
});

console.log(run.id, run.status); // run_abc123 queued
```

## Self-host setup

```bash
git clone https://github.com/strait-dev/strait.git
cd strait
make selfhost
```

That boots PostgreSQL, Redis, the Strait API, and the dashboard via Docker Compose. Open [http://localhost:3000](http://localhost:3000) and sign up.

The full self-host walkthrough — production configuration, hardening, observability — lives in [SELFHOST.md](https://github.com/strait-dev/strait/blob/master/SELFHOST.md). For a step-by-step development setup, see the [Quick Start](/quickstart).
