## Two Ways to Run Strait

Strait runs as a managed cloud service or self-hosted on your infrastructure. Both use the same API, SDKs, and features. Pick the option that fits your team's requirements.

| | Cloud | Self-Host |
|---|---|---|
| Setup | Sign up at app.strait.dev | Docker Compose or Kubernetes |
| Infrastructure | Managed by Strait | You manage PostgreSQL + Redis |
| Updates | Automatic | You pull and deploy |
| Pricing | Free tier + paid plans | Free forever |
| Data | Hosted by Strait | Your servers, your data |
| Support | Email + SLA (paid plans) | Community + GitHub Issues |
| Best for | Teams that want to start fast | Teams with compliance or data requirements |

## Cloud Quickstart

<Steps>
  <Step title="Sign up">
    Create an account at [app.strait.dev](https://app.strait.dev).
  </Step>
  <Step title="Get your API key">
    Go to **Settings > API Keys** and create a new key.
  </Step>
  <Step title="Install an SDK and trigger your first job">
    ```typescript
    import { createClient } from "@strait/ts";

    const client = createClient({
      baseUrl: "https://api.strait.dev",
      auth: {
        type: "bearer",
        token: "sk_live_...",
      },
    });

    // Trigger a job
    const run = await client.triggerJob({
      pathParams: { jobID: "my-first-job" },
      body: { payload: { message: "Hello from Strait!" } },
    });

    console.log(run.id, run.status); // run_abc123 queued
    ```
  </Step>
</Steps>

## Self-Host Quickstart

<Steps>
  <Step title="Clone and start">
    ```bash
    git clone https://github.com/strait-dev/strait.git
    cd strait
    make selfhost
    ```
    This starts Strait with PostgreSQL and Redis via Docker Compose.
  </Step>
  <Step title="Open the dashboard">
    Go to [http://localhost:3000](http://localhost:3000) to access the Strait dashboard.
  </Step>
  <Step title="Next steps">
    Build from source or configure production settings: [Full quickstart guide](/quickstart).
  </Step>
</Steps>
