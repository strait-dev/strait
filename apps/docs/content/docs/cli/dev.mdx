The Go CLI is config-driven. Project state is sourced from `strait.json` or `strait.config.yaml`/`.yml`, compiled into a deterministic manifest, then promoted through deployment versions.

## build

Compile local definitions into a deterministic manifest payload.

| Flag | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `--config` | string | auto-discover config file | Config path override |
| `--out-dir` | string | `.strait` or config value | Manifest output directory override |
| `--dry-run` | bool | false | Print manifest payload without writing files |
| `--json` | bool | false | Emit manifest JSON to stdout without writing files |

```bash
strait build --dry-run --json
```

## dev

Start the local runtime or inspect local readiness.

Commands:
- `strait dev`
- `strait dev --no-docker`
- `strait dev status`

## deploy

Create and optionally finalize a deployment version from a compiled manifest.

Commands:
- `strait deploy --config strait.config.yaml --artifact-uri https://example.com/artifact.tgz`
- `strait deploy create --config strait.config.yaml --artifact-uri https://example.com/artifact.tgz`
- `strait deploy finalize dep_123 --project proj_123`
- `strait deploy promote dep_123 --project proj_123 --env production`
- `strait deploy rollback --to dep_101 --project proj_123 --env production`

Common flags:
- `--config`
- `--env`
- `--artifact-uri`
- `--project`
- `--dry-run`

## Removed assumptions

These pages document the shipped Go CLI. They do not describe a separate TypeScript CLI or `@strait/cli` package.
