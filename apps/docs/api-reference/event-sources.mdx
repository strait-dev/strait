Use Event Sources when an external system should send events into Strait.

For request body fields generated from the runtime OpenAPI schema, see [Generated API Fields](/api-reference/generated-fields#event-sources).

## Prerequisites

```bash
export STRAIT_BASE_URL=https://api.strait.dev
export STRAIT_API_KEY=strait_replace_with_your_key
```

## Create an Event Source

```bash
curl -sS -X POST "$STRAIT_BASE_URL/v1/event-sources" \
  -H "Authorization: Bearer $STRAIT_API_KEY" \
  -H "Content-Type: application/json" \
  --data @apps/docs/examples/requests/create-event-source.json | jq
```

Expected result: an event source with an ID.

## Dispatch an Event

```bash
curl -sS -X POST "$STRAIT_BASE_URL/v1/events/dispatch" \
  -H "Authorization: Bearer $STRAIT_API_KEY" \
  -H "Content-Type: application/json" \
  --data @apps/docs/examples/requests/dispatch-event.json | jq
```

## Common Operations

| Operation | Method and path |
|---|---|
| List event sources | `GET /v1/event-sources` |
| Create event source | `POST /v1/event-sources` |
| Get event source | `GET /v1/event-sources/{sourceID}` |
| Update event source | `PATCH /v1/event-sources/{sourceID}` |
| Delete event source | `DELETE /v1/event-sources/{sourceID}` |
| Subscribe target | `POST /v1/event-sources/{sourceID}/subscribe` |
| List event source subscriptions | `GET /v1/event-sources/{sourceID}/subscriptions` |
| Delete subscription | `DELETE /v1/event-sources/{sourceID}/subscriptions/{subID}` |
| Dispatch event | `POST /v1/events/dispatch` |
| List event triggers | `GET /v1/events` |
| Get event trigger statistics | `GET /v1/events/stats` |
| Purge event triggers | `POST /v1/events/purge` |
| Send event by prefix | `POST /v1/events/prefix/{prefix}/send` |
| Get event trigger | `GET /v1/events/{eventKey}` |
| Cancel event trigger | `DELETE /v1/events/{eventKey}` |
| Send event key | `POST /v1/events/{eventKey}/send` |
| Stream event trigger updates | `GET /v1/events/{eventKey}/stream` |

## Failure Cases

| Status | Meaning |
|---|---|
| `400` | Source, signature config, or payload is invalid. |
| `401` | API key missing or invalid. |
| `404` | Source, subscription, or trigger does not exist. |
| `409` | Duplicate source or subscription conflict. |
