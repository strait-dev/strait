This page lists the settings most deployments need. The full runtime has additional tuning flags, but most teams should start here.

## Required

| Variable | Description |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string. |
| `REDIS_URL` | Redis connection string. Required unless `REDIS_SENTINEL_MASTER` and `REDIS_SENTINEL_ADDRS` are configured. |
| `SEQUIN_BASE_URL` | Sequin API base URL. |
| `SEQUIN_CONSUMER_NAME` | Sequin HTTP pull consumer name. |
| `SEQUIN_API_TOKEN` | Sequin API token provisioned in the Sequin config. |
| `INTERNAL_SECRET` | Secret for trusted service-to-service access. Minimum 16 characters; 32 or more recommended. |
| `JWT_SIGNING_KEY` | Secret used to sign run tokens for SDK calls. 32 or more characters recommended. |
| `ENCRYPTION_KEY` | Key used for encrypted data at rest. |
| `SECRET_ENCRYPTION_KEY` | Key used to encrypt stored secrets. |
| `STRAIT_ENV` | Deployment environment for runtime safety checks. Use `development` or `test` only outside production-like deployments. |

Example:

```bash
export DATABASE_URL="postgres://strait:strait@localhost:15432/strait?sslmode=disable"
export REDIS_URL="redis://localhost:16379"
export SEQUIN_BASE_URL="http://localhost:7376"
export SEQUIN_CONSUMER_NAME="strait-cdc"
export SEQUIN_API_TOKEN="$(openssl rand -hex 32)"
export SEQUIN_WEBHOOK_SECRET="$(openssl rand -hex 32)"
export INTERNAL_SECRET="$(openssl rand -base64 32)"
export JWT_SIGNING_KEY="$(openssl rand -base64 32)"
export ENCRYPTION_KEY="$(openssl rand -base64 32)"
export SECRET_ENCRYPTION_KEY="$(openssl rand -base64 32)"
export STRAIT_ENV="production"
```

## Recommended

| Variable | Default | Description |
|---|---|---|
| `MODE` | `all` | `all`, `api`, or `worker`. |

| `LOG_LEVEL` | `info` | `debug`, `info`, `warn`, or `error`. |

## Worker Settings

| Variable | Default | Description |
|---|---|---|
| `WORKER_CONCURRENCY` | `25` | Max jobs a worker can execute at the same time. |
| `WORKER_QUEUE_SIZE` | `0` | Local queue size per worker. `0` means unbounded. |
| `EXECUTOR_HTTP_TIMEOUT` | `5m` | Max time for HTTP job execution. |
| `EXECUTION_TRACE_MODE` | `off` | Persist per-run execution trace JSON. Use `off`, `errors`, or `full`. |
| `MAX_DEQUEUE_BATCH_SIZE` | `50` | Max runs one worker polls from PgQue at once. |
| `DEFAULT_JOB_TIMEOUT_SECS` | `300` | Default timeout for jobs without an explicit timeout. |
| `DEFAULT_JOB_MAX_ATTEMPTS` | `3` | Default retry attempts for jobs without an explicit value. |

## Cache Settings

Redis is still required at startup. Cache operations fail open to in-process cache or PostgreSQL after startup, except authentication paths confirm with PostgreSQL when freshness is uncertain.

| Variable | Default | Description |
|---|---|---|
| `JOB_CACHE_TTL` | `5m` | Current job definition cache TTL for worker dispatch. Set to `0` to disable. |
| `VERSION_CACHE_TTL` | `30m` | Immutable job and workflow definition cache TTL. |
| `RUN_VERSION_CACHE_TTL` | `10m` | TTL for immutable workflow run version lookups used by worker execution policy resolution. |
| `API_KEY_CACHE_TTL` | `60s` | API key auth cache TTL. Revocation and rotation invalidate immediately; cached values exclude secret material. |
| `JOB_HEALTH_CACHE_TTL` | `30s` | Short TTL for adaptive job health aggregates used on hot worker paths. |
| `ENDPOINT_HEALTH_SUCCESS_SAMPLE_INTERVAL` | `1s` | Minimum interval between successful endpoint health writes per worker and endpoint. Failures always write immediately. |
| `ENDPOINT_CIRCUIT_SUCCESS_SAMPLE_INTERVAL` | `1s` | Minimum interval between successful endpoint circuit writes per worker and endpoint. Failures always write immediately. |
| `JOB_DEPS_CACHE_TTL` | `5m` | Job dependency list cache TTL. Dependency create and delete invalidate affected entries. |
| `CACHE_STATUS_READMODEL_TTL` | `5m` | Redis status read-model TTL for job runs and workflow runs. CDC updates use version compare-and-set so status does not move backward. |
| `CACHE_SHARED_DEDUPE_TTL` | `10m` | Redis duplicate suppression TTL for CDC side effects and Sequin push webhooks. |
| `PERMISSION_CACHE_TTL` | `5m` | RBAC permission cache TTL. |
| `PROJECT_QUOTA_CACHE_TTL` | `60s` | Project quota cache TTL. |

## Queue

Strait uses a vendored and modified SQL snapshot of [PgQue](https://github.com/NikolayS/pgque) as its PostgreSQL ready-event log. Strait owns run state, execution ownership, retries, workflows, workers, observability, and APIs.

| Variable | Default | Description |
|---|---|---|
| `QUEUE_PGQUE_MAINTENANCE_INTERVAL` | `30s` | PgQue maintenance interval. Strait runs PgQue maintenance from its ticker so ready-event tables rotate without operator-managed cron. |
| `QUEUE_PGQUE_ROTATION_PERIOD` | `5m` | PgQue event-table rotation period for Strait queue routes. Shorter periods reclaim acknowledged ready-event rows sooner. |

| Variable | Default | Description |
|---|---|---|
| `RATE_LIMIT_REQUESTS` | `100` | Global requests allowed per window. |
| `RATE_LIMIT_WINDOW` | `1m` | Global rate-limit window. |
| `TRIGGER_RATE_LIMIT_REQUESTS` | `10` | Trigger requests allowed per window. |
| `TRIGGER_RATE_LIMIT_WINDOW` | `1m` | Trigger rate-limit window. |
| `MAX_BULK_TRIGGER_ITEMS` | `500` | Max items in one bulk trigger request. |
| `BACKPRESSURE_ENABLED` | `true` | Enable the per-project enqueue token bucket. Keep enabled unless an upstream limiter enforces equivalent tenant isolation. |
| `BACKPRESSURE_DEFAULT_MAX_TOKENS` | `1000` | Initial enqueue burst capacity for projects without an explicit project rate-limit row. |
| `BACKPRESSURE_DEFAULT_REFILL_PER_SEC` | `500` | Steady-state accepted enqueue refill rate per project for projects without an explicit project rate-limit row. |
| `BACKPRESSURE_LOCAL_LEASE_SIZE` | `32` | DB-backed token lease size reserved per process before serving single-run admissions from memory. Set to `1` for strict per-trigger DB accounting. |
| `DB_BACKPRESSURE_DISABLED` | `false` | Disable API database admission backpressure. Use only for controlled capacity tests or when an upstream limiter protects Postgres. |
| `DB_BACKPRESSURE_SAMPLE_INTERVAL` | `100ms` | Interval for sampling Postgres pool acquire-wait pressure. |
| `DB_BACKPRESSURE_ACQUIRE_WAIT_THRESHOLD` | `50ms` | Average connection acquire wait that starts database admission shedding. |
| `DB_BACKPRESSURE_OCCUPANCY_THRESHOLD` | `0.90` | Pool occupancy ratio that starts database admission shedding. |

## Monitoring

| Variable | Default | Description |
|---|---|---|
| `OTEL_EXPORTER_OTLP_ENDPOINT` | - | OpenTelemetry endpoint for traces and logs. |
| `OTEL_LOGS_EXPORTER` | `none` recommended locally | Set to `none` for local development when no OTLP logs endpoint is available, otherwise the collector may emit repeated `/v1/logs` errors. |
| `OTLP_METRIC_ENDPOINT` | - | OpenTelemetry endpoint for metrics, if different. |
| `OTLP_METRIC_ENABLED` | `false` | Enable OTLP metric export. |
| `DB_TRACE_STATEMENTS` | `false` | Enable per-SQL-statement pgx tracing and Sentry breadcrumbs. Keep disabled on hot production paths unless diagnosing database behavior. |
| `SENTRY_DSN` | - | Sentry DSN for error reporting. |
| `SENTRY_ENVIRONMENT` | - | Environment tag for Sentry and OTel resources, such as `production` or `staging`. |
| `SENTRY_TRACES_SAMPLE_RATE` | `0.1` | Sentry trace sample rate from `0.0` to `1.0`. Tracing is enabled when this is greater than zero, and high-volume stream transactions are dropped by the Sentry sampler before send. |
| `SENTRY_RELEASE` | - | Release identifier attached to Sentry events. Defaults to the binary version when unset. Use the same value for frontend sourcemap uploads. |
| `SENTRY_DEBUG` | `false` | Enable Sentry SDK debug logging for troubleshooting. Keep disabled in production unless actively diagnosing ingestion. |
| `SENTRY_MAX_BREADCRUMBS` | `100` | Maximum breadcrumbs attached to an event. Set negative to disable breadcrumbs. |
| `SENTRY_MAX_SPANS` | `1000` | Maximum spans attached to a Sentry transaction. |
| `SENTRY_MAX_ERROR_DEPTH` | `100` | Maximum wrapped-error chain depth reported by the Sentry SDK. |
| `SENTRY_STRICT_TRACE_CONTINUATION` | `false` | Require incoming Sentry baggage org IDs to match before continuing third-party traces. |
| `SENTRY_SCHEDULER_CHECKINS` | `false` | Emit Sentry monitor check-ins for scheduler components when enabled. |
| `SENTRY_SCHEDULER_CHECKIN_PREFIX` | `strait-scheduler` | Prefix used for scheduler component monitor slugs. |
| `STRAIT_PROFILING_ENABLED` | `false` | Expose internal-secret-protected Go pprof endpoints under `/debug/pprof`. |
| `STRAIT_PROFILING_API_ENABLED` | `true` | Expose pprof on the API listener when profiling is enabled. |
| `STRAIT_PROFILING_MANAGEMENT_ENABLED` | `false` | Start a separate listener that serves only pprof routes. |
| `STRAIT_PROFILING_MANAGEMENT_BIND_ADDR` | `127.0.0.1` | Bind address for the pprof management listener. |
| `STRAIT_PROFILING_MANAGEMENT_PORT` | `18080` | Port for the pprof management listener. |
| `STRAIT_PROFILING_MUTEX_FRACTION` | `100` | Go runtime mutex profile sampling fraction. `0` disables mutex profiling after restart. |
| `STRAIT_PROFILING_BLOCK_RATE` | `100000` | Go runtime block profile sampling rate. `0` disables block profiling after restart. |
| `STRAIT_PROFILING_SECRET` | - | Optional pprof-specific secret. When set, pprof requires this secret instead of `INTERNAL_SECRET`. |
| `STRAIT_PROFILING_ALLOWED_CIDRS` | - | Optional comma-separated CIDR/IP allowlist for pprof callers. |
| `PYROSCOPE_ENDPOINT` | - | Pyroscope endpoint for profiling. |

## Worker DLQ

| Variable | Default | Description |
|---|---|---|
| `DLQ_MAX_PER_PROJECT` | `10000` | Maximum visible dead-letter runs per project. Set `0` to disable the project cap. |
| `DLQ_MAX_PER_JOB` | `1000` | Maximum visible dead-letter runs per job. Set `0` to disable the job cap. |
| `DLQ_OVERFLOW_POLICY` | `drop_oldest` | `drop_oldest` masks the oldest visible DLQ row before adding a new one. `reject` marks the new terminal failure `system_failed` with `error_class=dlq_overflow`. |

## Workflow Recovery

| Variable | Default | Description |
|---|---|---|
| `WF_STALL_THRESHOLD` | `15m` | How long a running workflow can make no progress before the reaper treats it as stalled. |
| `WF_STALL_ACTION` | `reconcile` | Action for stalled workflows. `reconcile` resumes persisted workflow state, `log_only` only logs, and `fail_workflow` marks the workflow failed. |

## Worker Mode

Use these when running connected workers.

| Variable | Default | Description |
|---|---|---|
| `GRPC_BIND_ADDR` | `127.0.0.1` | Address for the gRPC server. |
| `GRPC_PORT` | `50051` | gRPC server port. |
| `GRPC_ALLOW_PLAINTEXT` | `false` | Allow plaintext gRPC on non-loopback addresses. |
| `GRPC_TLS_CERT_PATH` | - | TLS certificate path. |
| `GRPC_TLS_KEY_PATH` | - | TLS private key path. |
| `WORKER_DB_SYNC_INTERVAL` | `15s` | Interval at which connected worker state is flushed to the database. Must be greater than `HEARTBEAT_INTERVAL` and less than `STALE_THRESHOLD`. |
| `WORKER_DISCONNECT_ACK_TIMEOUT` | `5s` | Maximum time `DELETE /workers/{id}` waits for worker-plane disconnect acknowledgement before returning `503` with `Retry-After`. |
| `GRPC_PUBSUB_STARTUP_TIMEOUT` | `30s` | Maximum time gRPC startup waits for pubsub readiness before refusing to serve worker streams. |
| `EXTERNAL_API_URL` | - | Base URL workers use to call back to the Strait API. |

## Sizing

| Variable | Default | Description |
|---|---|---|
| `DB_MAX_CONNS` | `50` | Maximum PostgreSQL pool size per Strait process. Keep this above `WORKER_CONCURRENCY` and multiply by replicas when sizing PostgreSQL. |
| `DB_MIN_CONNS` | `10` | Minimum warm PostgreSQL connections per Strait process. |
| `DB_PGBOUNCER_MODE` | `false` | Use simple protocol for PgBouncer transaction pooling when prepared statements are not available. |
| `DB_PGBOUNCER_PREPARED_STATEMENTS` | `false` | Keep pgx prepared statement caching on when PgBouncer 1.21 or newer has `max_prepared_statements` configured. |
| `REDIS_POOL_SIZE` | `30` | Maximum Redis connections per Strait process. |
| `REDIS_MIN_IDLE_CONNS` | `5` | Minimum warm Redis connections per Strait process. |
| `WORKER_CONCURRENCY` | `25` | Maximum concurrent job executions per worker process. |
