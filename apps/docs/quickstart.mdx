This guide gets you to one completed run. The default path uses Strait Cloud. Community users can run the same API commands after exporting a local `STRAIT_BASE_URL`, `STRAIT_API_KEY`, and `STRAIT_PROJECT_ID`.

## Prerequisites

For Cloud:

- A Strait Cloud account at [app.strait.dev](https://app.strait.dev)
- A project ID
- An API key for that project
- `curl`
- `jq`

For community local setup, complete [Community Local Setup](/choose-your-path#community-local-setup) first.

## 1. Export Connection Settings

Cloud:

```bash
export STRAIT_BASE_URL=https://api.strait.dev
export STRAIT_API_KEY=strait_replace_with_your_key
export STRAIT_PROJECT_ID=proj_replace_with_your_project_id
```

Community local:

```bash
export STRAIT_BASE_URL=http://localhost:8080
export STRAIT_API_KEY=my-dev-secret-change-in-production-min-32-chars
export STRAIT_PROJECT_ID=proj_replace_with_your_project_id
```

Check that the API is reachable:

```bash
curl -sS "$STRAIT_BASE_URL/health" | jq
```

Expected response:

```json
{
  "status": "ok",
  "edition": "cloud"
}
```

A community instance returns its own edition value.

## 2. Create A Job

This job posts to `https://httpbin.org/post`, which returns a successful response for the first test run. Use your own endpoint for real work.

```bash
curl -sS -X POST "$STRAIT_BASE_URL/v1/jobs" \
  -H "Authorization: Bearer $STRAIT_API_KEY" \
  -H "Content-Type: application/json" \
  -d "{
    \"project_id\": \"$STRAIT_PROJECT_ID\",
    \"name\": \"Hello World\",
    \"slug\": \"hello-world\",
    \"endpoint_url\": \"https://httpbin.org/post\",
    \"timeout_secs\": 60,
    \"max_attempts\": 3
  }" | tee /tmp/strait-job.json | jq
```

Save the job ID:

```bash
export STRAIT_JOB_ID=$(jq -r '.id' /tmp/strait-job.json)
```

## 3. Trigger A Run

```bash
curl -sS -X POST "$STRAIT_BASE_URL/v1/jobs/hello-world/trigger" \
  -H "Authorization: Bearer $STRAIT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "payload": {
      "greeting": "Hello from Strait"
    },
    "tags": {
      "source": "quickstart"
    }
  }' | tee /tmp/strait-run.json | jq
```

Save the run ID:

```bash
export STRAIT_RUN_ID=$(jq -r '.id' /tmp/strait-run.json)
```

The initial status is usually `queued` or `executing`.

## 4. Inspect The Run

Give the worker a few seconds, then read the run:

```bash
sleep 3
curl -sS "$STRAIT_BASE_URL/v1/runs/$STRAIT_RUN_ID" \
  -H "Authorization: Bearer $STRAIT_API_KEY" | jq
```

Expected fields:

```json
{
  "id": "run_abc123",
  "job_id": "job_abc123",
  "status": "completed",
  "attempt": 1
}
```

If the run is still `queued` or `executing`, repeat the same `GET /v1/runs/{runID}` command.

## 5. Replay The Run

Replay creates a new run from the same work after you fix an endpoint or dependency problem. You can also replay a successful quickstart run to verify the route.

```bash
curl -sS -X POST "$STRAIT_BASE_URL/v1/runs/$STRAIT_RUN_ID/replay" \
  -H "Authorization: Bearer $STRAIT_API_KEY" | jq
```

## Common Problems

| Problem | Fix |
|---|---|
| `401 Unauthorized` | Use the project API key as `STRAIT_API_KEY`. Do not use the dashboard password. |
| `403 Forbidden` | Confirm `STRAIT_PROJECT_ID` belongs to the API key's project. |
| `400 endpoint_url failed validation` | Use a public HTTPS endpoint for Cloud. Cloud rejects private IP ranges and metadata endpoints. |
| Run stays `queued` on community | Start Strait with API, scheduler, and worker enabled. Local `make dev` starts the full stack. |
| Run moves to `dead_letter` | Read the run error, fix the endpoint, then replay. |
