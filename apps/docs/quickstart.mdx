This guide gets you to one completed run. After that, you can add retries, schedules, workflows, SDKs, and production settings.

If you use Strait Cloud, use `https://api.strait.dev` as the base URL and your API key as the bearer token. You can skip the local setup steps.

## What You Will Do

1. Start Strait locally.
2. Create a project.
3. Create a job.
4. Trigger a run.
5. Inspect the result.

## Prerequisites

You need:

- Go 1.26+
- Docker with Compose v2
- `curl`

## 1. Start Strait

```bash
git clone https://github.com/strait-dev/strait.git
cd strait

docker compose -f apps/strait/docker-compose.yml up --build --wait -d
docker compose -f apps/strait/docker-compose.yml ps
```

PostgreSQL, Redis, Sequin, and Strait should report healthy.

## 2. Configure Strait

```bash
export DATABASE_URL=postgres://strait:strait@localhost:15432/strait?sslmode=disable
export REDIS_URL=redis://localhost:16379
export SEQUIN_BASE_URL=http://localhost:7376
export SEQUIN_CONSUMER_NAME=strait-cdc
export INTERNAL_SECRET=my-dev-secret-change-in-production-min-32-chars
export JWT_SIGNING_KEY=my-jwt-key-change-in-production-min-32-chars
```

For local development, `INTERNAL_SECRET` is the bearer token for API calls.

## 3. Run the Server From Source

```bash
docker compose -f apps/strait/docker-compose.yml stop strait
cd apps/strait
go build -o strait ./cmd/strait
./strait --mode all
```

The Compose stack already runs Strait. Stop the Compose `strait` container only when you want to run the local binary against the same PostgreSQL, Redis, and Sequin services. Mode `all` runs the API, scheduler, and worker in one process.

## 4. Create a Project

```bash
curl -X POST http://localhost:8080/v1/projects \
  -H "Authorization: Bearer $INTERNAL_SECRET" \
  -H "Content-Type: application/json" \
  -d '{ "name": "My Project", "slug": "my-project" }'
```

Save the returned project ID.

## 5. Create a Job

A job defines where Strait sends work, how long it may run, and how many times it may retry.

```bash
curl -X POST http://localhost:8080/v1/jobs \
  -H "Authorization: Bearer $INTERNAL_SECRET" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "proj_abc123",
    "name": "Hello World",
    "slug": "hello-world",
    "endpoint_url": "https://httpbin.org/post",
    "timeout_secs": 60,
    "max_attempts": 3
  }'
```

Replace `proj_abc123` with your project ID.

## 6. Trigger a Run

```bash
curl -X POST http://localhost:8080/v1/jobs/hello-world/trigger \
  -H "Authorization: Bearer $INTERNAL_SECRET" \
  -H "Content-Type: application/json" \
  -d '{ "payload": { "greeting": "Hello from Strait" } }'
```

The response includes a run ID and a status such as `queued`.

## 7. Inspect the Run

```bash
curl http://localhost:8080/v1/runs/run_abc123 \
  -H "Authorization: Bearer $INTERNAL_SECRET"
```

A successful run moves to `completed`. The run record shows the payload, attempts, timing, status, and result.

## Common Problems

**Run stays queued.** Make sure Strait is running with `--mode all` or `--mode worker`.

**Endpoint times out.** Increase `timeout_secs` or make the endpoint finish faster.

**Private endpoint is rejected.** Strait blocks private IP ranges by default. See [Security](/guides/security#ssrf-protection).

**Run moves to `dead_letter`.** Fix the underlying issue, then replay the run. See [Handle Failed Runs](/guides/handle-failed-runs).
