## What is a Project?

A project is the top-level container in Strait. Every job, workflow, run, environment, secret, and API key belongs to exactly one project. Use projects to separate concerns — one per service, team, or deployment stage.

Projects provide:

- **Isolation** — jobs and workflows in one project cannot reference resources in another.
- **Quotas** — per-project limits on concurrency, cost, and rate of API calls.
- **Scoped API keys** — each key grants access to a single project with fine-grained permission scopes.

---

## Creating a Project

```bash
curl -X POST https://api.yourproject.strait.sh/v1/projects \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Payments Service",
    "org_id": "org_acme"
  }'
```

Response:

```json
{
  "id": "proj_abc123",
  "org_id": "org_acme",
  "name": "Payments Service",
  "created_at": "2025-06-01T12:00:00Z",
  "updated_at": "2025-06-01T12:00:00Z"
}
```

A project has five fields: `id`, `org_id`, `name`, `created_at`, and `updated_at`.

## Organizations

Projects can belong to an organization via the `org_id` field. An organization is a string identifier that groups projects for billing and cross-project queries. Set `org_id` when creating a project to associate it with an organization.

API keys can also carry an `org_id`, granting access across all projects in that organization instead of a single project.

## Quotas and Limits

Every project has a quota that caps concurrency, throughput, and cost. View your project's current limits:

```bash
curl https://api.yourproject.strait.sh/v1/projects/proj_abc123/settings \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Key quota fields:

| Field | Description |
|---|---|
| `max_queued_runs` | Maximum runs waiting in the queue. |
| `max_executing_runs` | Maximum runs executing concurrently. |
| `max_jobs` | Maximum job definitions in the project. |
| `max_workflows` | Maximum workflow definitions in the project. |
| `max_cost_per_run_microusd` | Cost ceiling per individual run, in micro-USD. |
| `max_daily_cost_microusd` | Total daily cost ceiling for the project, in micro-USD. |
| `rate_limit_requests` | Maximum API requests per second. |

Quotas are tied to your `plan_tier`. When a limit is reached, Strait rejects the operation with a clear error rather than silently dropping work.

## API Key Scopes

Each API key is scoped to a project and carries a set of permission scopes. Assign only the scopes a service needs — a worker that triggers jobs does not need `secrets:write`.

| Category | Scopes |
|---|---|
| **Jobs** | `jobs:read`, `jobs:write`, `jobs:trigger` |
| **Runs** | `runs:read`, `runs:write` |
| **Workflows** | `workflows:read`, `workflows:write`, `workflows:trigger` |
| **Environments** | `environments:read`, `environments:write` |
| **Secrets** | `secrets:read`, `secrets:write` |
| **API Keys** | `api-keys:manage` |
| **Webhooks** | `webhooks:read`, `webhooks:write` |
| **dead_letter runs** | `dlq:read`, `dlq:replay`, `dlq:purge` |
| **Analytics** | `analytics:read` |
| **Audit** | `audit:read` |
| **RBAC** | `rbac:manage` |
| **Workers** | `workers:connect` |
| **Projects** | `projects:read`, `projects:write`, `projects:manage` |

## How Resources Relate

```
Organization (org_id)
  └── Project
        ├── Jobs
        ├── Workflows
        ├── Environments
        ├── Secrets
        └── API Keys
```

Everything lives inside a project. Organizations group projects but do not own resources directly — they exist for billing aggregation and cross-project API key access.
