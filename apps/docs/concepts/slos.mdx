## Service Level Objectives

SLOs let you define reliability targets per job. Strait tracks three metrics:

| Metric | What it measures |
|---|---|
| `success_rate` | Percentage of runs that complete without error |
| `p95_latency_secs` | 95th percentile run duration in seconds |
| `p99_latency_secs` | 99th percentile run duration in seconds |

Each SLO is evaluated over a configurable window: **24 hours**, **7 days** (168h), or **30 days** (720h).

## Creating an SLO

```bash
curl -X POST https://api.strait.so/v1/jobs/{jobID}/slos \
  -H "Authorization: Bearer $STRAIT_API_KEY" \
  -d '{
    "metric": "success_rate",
    "target": 0.995,
    "window_hours": 24
  }'
```

```json
{
  "id": "slo_abc123",
  "job_id": "job_xyz",
  "project_id": "proj_456",
  "metric": "success_rate",
  "target": 0.995,
  "window_hours": 24
}
```

You can create multiple SLOs per job — one for success rate and another for latency, for example.

## Evaluating SLOs

Check how a job is performing against its targets:

```bash
curl https://api.strait.so/v1/jobs/{jobID}/slos/evaluate \
  -H "Authorization: Bearer $STRAIT_API_KEY"
```

```json
{
  "evaluations": [
    {
      "id": "eval_001",
      "slo_id": "slo_abc123",
      "current_value": 0.997,
      "budget_remaining": 0.002,
      "evaluated_at": "2025-01-15T10:00:00Z"
    }
  ]
}
```

`budget_remaining` tells you how much room you have before breaching the target. A `success_rate` SLO with target `0.995` and current value `0.997` has `0.002` of error budget left.

## Deleting an SLO

```bash
curl -X DELETE https://api.strait.so/v1/jobs/{jobID}/slos/{sloID} \
  -H "Authorization: Bearer $STRAIT_API_KEY"
```

## Anomaly Detection

Strait monitors spending patterns across your usage and flags unusual spikes. Two severity levels:

- **Warning** — spending exceeded the warning threshold compared to the recent baseline
- **Critical** — spending exceeded the critical threshold

## Configuring anomaly detection

Set your thresholds with a multiplier. A `warning_threshold` of `1.5` means spending 1.5x above the baseline triggers a warning.

```bash
curl -X PUT https://api.strait.so/v1/anomaly-config \
  -H "Authorization: Bearer $STRAIT_API_KEY" \
  -d '{
    "warning_threshold": 1.5,
    "critical_threshold": 3.0
  }'
```

Retrieve current configuration:

```bash
curl https://api.strait.so/v1/anomaly-config \
  -H "Authorization: Bearer $STRAIT_API_KEY"
```

## Checking for anomalies

```bash
curl https://api.strait.so/v1/usage/anomalies \
  -H "Authorization: Bearer $STRAIT_API_KEY"
```

```json
{
  "anomalies": [
    {
      "detected_at": "2025-01-15T08:30:00Z",
      "severity": "warning",
      "current_spend": 42.50,
      "baseline_spend": 25.00,
      "ratio": 1.7
    }
  ]
}
```

An empty `anomalies` array means spending is within normal bounds.
