## Service Level Objectives

Strait tracks reliability signals from workflow and run execution and exposes them so you can define SLO targets and alerts in your own monitoring stack. Strait does not evaluate SLO targets or fire breach alerts itself; it provides the signals, and you set the thresholds in Prometheus, Grafana, or whatever you already run.

| Signal | What it measures |
|---|---|
| `success_rate` | Percentage of runs that complete without error |
| `p95_latency_secs` | 95th percentile run duration in seconds |
| `p99_latency_secs` | 99th percentile run duration in seconds |

Query these signals over a window with the `period_hours` parameter, which accepts 1 to 720 hours (up to 30 days) and defaults to 24:

```bash
curl "https://api.strait.dev/v1/analytics/performance?period_hours=168" \
  -H "Authorization: Bearer $STRAIT_API_KEY"
```

## Setting Targets and Alerts

The same signals are exported as Prometheus metrics at `GET /metrics`. Define your SLO target (for example, 99.5% success over 30 days) as an alerting rule in Prometheus or Grafana against those metrics, and route the breach notification through your existing alerting. See [Observability](/operations/observability) for the metric catalog and scrape setup.

This keeps SLO ownership where the rest of your alerting already lives, rather than splitting it across another product's thresholds.

## API Surface

| Endpoint | Purpose |
|---|---|
| `GET /v1/analytics/performance` | Review run duration, throughput, and success trends (`period_hours` 1-720). |
| `GET /v1/usage/anomalies` | List detected usage and spend anomalies. |
| `GET /v1/anomaly-config` | Read anomaly detection thresholds. |
| `PUT /v1/anomaly-config` | Update anomaly detection thresholds. |
| `GET /metrics` | Export Prometheus metrics for external alerting. |

## Anomaly Detection

Strait monitors spending patterns across your usage and flags unusual spikes. Two severity levels:

- **Warning**: spending exceeded the warning threshold compared to the recent baseline.
- **Critical**: spending exceeded the critical threshold.

## Configuring anomaly detection

Set your thresholds with a multiplier. A `warning_threshold` of `1.5` means spending 1.5x above the baseline triggers a warning.

```bash
curl -X PUT https://api.strait.dev/v1/anomaly-config \
  -H "Authorization: Bearer $STRAIT_API_KEY" \
  -d '{
    "warning_threshold": 1.5,
    "critical_threshold": 3.0
  }'
```

Retrieve current configuration:

```bash
curl https://api.strait.dev/v1/anomaly-config \
  -H "Authorization: Bearer $STRAIT_API_KEY"
```

## Checking for anomalies

```bash
curl https://api.strait.dev/v1/usage/anomalies \
  -H "Authorization: Bearer $STRAIT_API_KEY"
```

```json
{
  "anomalies": [
    {
      "detected_at": "2026-01-15T08:30:00Z",
      "severity": "warning",
      "current_spend": 42.50,
      "baseline_spend": 25.00,
      "ratio": 1.7
    }
  ]
}
```

An empty `anomalies` array means spending is within normal bounds.
