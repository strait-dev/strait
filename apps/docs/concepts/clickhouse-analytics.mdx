ClickHouse is an optional analytics backend for larger Strait deployments. PostgreSQL remains the source of truth. ClickHouse is used for reporting queries over large time windows.

You do not need ClickHouse to run Strait. It exists for deployments where aggregate reporting, cost breakdowns, or time-series dashboards matter at scale.

## When to Use It

Use ClickHouse when:

- You have millions of runs and need aggregate analytics (success rates, p99 durations, cost trends).
- You want pre-aggregated daily stats without loading your primary database.
- You need time-series dashboards (Grafana, Metabase) backed by a columnar store.

For smaller deployments, the default reporting path is usually enough. Turn on ClickHouse when the reporting workload justifies another service.

## How It Works

When enabled, Strait sends run and workflow events to ClickHouse for reporting. Job execution does not depend on ClickHouse being available.

## Reported Data

ClickHouse reporting includes:

- run counts by status, job, project, and time window
- duration percentiles and queue wait time
- workflow run and step timing
- webhook delivery history
- usage and cost trends
- approval and event-trigger timing

## Configuration

| Variable | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `CLICKHOUSE_ENABLED` | Boolean | `false` | Enable the ClickHouse client and create tables on startup. |
| `CLICKHOUSE_URL` | String | | Native protocol URL (e.g., `clickhouse://localhost:9000`). Required when enabled. |
| `CLICKHOUSE_DATABASE` | String | `strait` | Target database name. |
| `CLICKHOUSE_BATCH_SIZE` | Integer | `1000` | Maximum records per batch insert. |
| `CLICKHOUSE_FLUSH_INTERVAL` | Duration | `5s` | Maximum time between flushes. |
| `CLICKHOUSE_EXPORT_ENABLED` | Boolean | `false` | Enable the async exporter that sends analytics on run completion. |

<Warning>
Both `CLICKHOUSE_ENABLED` and `CLICKHOUSE_EXPORT_ENABLED` must be `true` for analytics data to flow. The first creates the schema; the second starts the exporter.
</Warning>

## Querying the Data

ClickHouse is a reporting store, not a second source of truth. Point Grafana, Metabase, or any ClickHouse-compatible client at `CLICKHOUSE_DATABASE` to build dashboards over the exported tables. Strait keeps serving runs from PostgreSQL regardless of whether the exporter is keeping up or the ClickHouse service is reachable.
