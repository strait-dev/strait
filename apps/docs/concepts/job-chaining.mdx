---
title: "Job Chaining"
description: "Trigger a follow-up job or workflow after a job completes or fails."
---

Job chaining connects simple sequences without creating a full workflow.

Use it when one job should start another job after it finishes.

## When to Use Chaining

Use chaining for short, linear flows:

```text
ingest-file -> normalize-file -> send-summary
```

Use [workflows](/concepts/workflows) when you need parallel steps, branching, approvals, waits, or a flow that several teams need to inspect.

If a chain grows beyond three or four steps, a workflow is usually easier to operate.

## Success and Failure Triggers

Configure the next action on the job:

| Field | Runs when |
| :--- | :--- |
| `on_complete_trigger_job` | This job completes successfully. |
| `on_complete_trigger_workflow` | This job completes successfully. |
| `on_failure_trigger_job` | This job reaches a terminal failure state. |
| `on_failure_trigger_workflow` | This job reaches a terminal failure state. |

A job can define both success and failure triggers.

## Payload Mapping

By default, the next job receives the completed job's result.

Use `on_complete_payload_mapping` to pass only the fields the next job needs.

```json
{
  "on_complete_trigger_job": "process-payment-result",
  "on_complete_payload_mapping": {
    "user_id": "user.id",
    "amount": "payment.amount"
  }
}
```

If the first job returns:

```json
{
  "user": { "id": "user_123" },
  "payment": { "amount": 49.99 }
}
```

The next job receives:

```json
{
  "user_id": "user_123",
  "amount": 49.99
}
```

## Failure Payloads

Failure triggers receive a payload with the failed run and error details.

```json
{
  "source_job_id": "job_123",
  "source_run_id": "run_456",
  "status": "failed",
  "attempt": 3,
  "error": "connection refused",
  "error_class": "connection",
  "original_input": {}
}
```

Use failure chains for notifications, cleanup, or a recovery job that can safely retry a separate path.

## Loop Protection

Strait limits chain depth to prevent accidental loops.

For example, if job A triggers job B and job B triggers job A, Strait stops the chain after the project limit is reached.
