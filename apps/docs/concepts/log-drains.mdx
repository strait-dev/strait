title: "Log Drains"
description: "Forward job execution logs to external platforms in real time."

Log drains forward your job execution logs to external platforms like Datadog, Splunk, or any HTTP endpoint in real time.

## Setting Up a Log Drain

Create a log drain that sends error and warning logs to Datadog:

```bash
curl -X POST "https://strait.example.com/v1/log-drains" \
  -H "Authorization: Bearer strait_..." \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "proj_1",
    "name": "datadog-errors",
    "drain_type": "http",
    "endpoint_url": "https://http-intake.logs.datadoghq.com/api/v2/logs",
    "auth_type": "header",
    "auth_config": {
      "DD-API-KEY": "your-datadog-api-key"
    },
    "level_filter": ["error", "warn"]
  }'
```

## Authentication

The `auth_type` field determines how Strait authenticates with the external endpoint:

| `auth_type` | `auth_config` keys | Behavior |
| :--- | :--- | :--- |
| `bearer` | `token` | Sends `Authorization: Bearer <token>`. |
| `basic` | `username`, `password` | Sends `Authorization: Basic <base64>`. |
| `header` | Any key-value pairs | Sends each entry as a custom HTTP header. |

<Warning>
The `auth_config` map contains sensitive credentials. Treat it with the same care as API keys or secrets.
</Warning>

## Filtering

The `level_filter` field accepts an array of log level strings. Only matching log events are forwarded. When omitted, all levels are delivered.

```json
{
  "level_filter": ["error", "warn"]
}
```

## Configuration

| Variable | Default | Description |
| :--- | :--- | :--- |
| `LOG_DRAIN_WORKER_INTERVAL` | `60s` | Interval at which the delivery worker polls for pending log events. |
