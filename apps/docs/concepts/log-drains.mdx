Log drains forward your job execution logs to external platforms like Datadog, Splunk, or any HTTP endpoint in real time.

In Strait Cloud, log drains are available on Starter and above. Launch caps are:

| Plan | Log drains |
| :--- | :--- |
| Free | 0 |
| Starter | 1 |
| Pro | 5 |
| Scale | 10 |
| Business | Unlimited |
| Enterprise | Unlimited |

## Setting Up a Log Drain

Create a log drain that sends error and warning logs to Datadog:

```bash
curl -X POST "https://your-strait-instance/v1/log-drains" \
  -H "Authorization: Bearer strait_..." \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "proj_abc123",
    "name": "datadog-errors",
    "drain_type": "http",
    "endpoint_url": "https://http-intake.logs.datadoghq.com/api/v2/logs",
    "auth_type": "header",
    "auth_config": {
      "DD-API-KEY": "your-datadog-api-key"
    },
    "level_filter": ["error", "warn"]
  }'
```

### Splunk HEC

```bash
curl -X POST "https://your-strait-instance/v1/log-drains" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "proj_abc123",
    "name": "splunk-errors",
    "drain_type": "http",
    "endpoint_url": "https://your-splunk:8088/services/collector/event",
    "auth_type": "header",
    "auth_config": {
      "Authorization": "Splunk your-hec-token"
    },
    "level_filter": ["error", "warn"]
  }'
```

### Generic HTTP Endpoint

```bash
curl -X POST "https://your-strait-instance/v1/log-drains" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "proj_abc123",
    "name": "custom-logs",
    "drain_type": "http",
    "endpoint_url": "https://your-service.com/logs",
    "auth_type": "bearer",
    "auth_config": {
      "token": "your-bearer-token"
    }
  }'
```

## Authentication

The `auth_type` field determines how Strait authenticates with the external endpoint:

| `auth_type` | `auth_config` keys | Behavior |
| :--- | :--- | :--- |
| `bearer` | `token` | Sends `Authorization: Bearer <token>`. |
| `basic` | `username`, `password` | Sends `Authorization: Basic <base64>`. |
| `header` | Any key-value pairs | Sends each entry as a custom HTTP header. |

<Warning>
The `auth_config` map contains sensitive credentials. Treat it with the same care as API keys or secrets.
</Warning>

## Filtering

The `level_filter` field accepts an array of log level strings. Only matching log events are forwarded. When omitted, all levels are delivered.

```json
{
  "level_filter": ["error", "warn"]
}
```

## Log Event Format

Each log event delivered to your drain endpoint is a JSON object:

```json
{
  "timestamp": "2026-03-15T10:05:23.456Z",
  "level": "error",
  "message": "HTTP POST to endpoint failed: connection refused",
  "run_id": "run_abc123",
  "job_id": "job_xyz",
  "project_id": "proj_abc123",
  "attempt": 2
}
```

Fields: timestamp (ISO 8601), level (debug/info/warn/error), message, run_id, job_id, project_id, attempt number.

## Configuration

| Variable | Default | Description |
| :--- | :--- | :--- |
| `LOG_DRAIN_WORKER_INTERVAL` | `60s` | Interval at which the delivery worker polls for pending log events. |

<Info>
Log drains use at-least-once delivery. Events are buffered and flushed on the configured interval. If delivery fails, events are retried. Your endpoint may receive duplicate events during retries. Use the `run_id` + `timestamp` combination for deduplication.
</Info>
