## When to use

A job is failing intermittently and logs alone don't explain why. Enable debug mode to capture events, checkpoints, progress, outputs, and resource utilization. Once the run completes, retrieve the bundle and inspect what happened.

## Enabling debug mode

Two ways to turn it on:

**At trigger time**: set `debug_mode` on the request that starts the run.

```bash
curl -X POST https://api.strait.dev/v1/jobs/{jobID}/trigger \
  -H "Authorization: Bearer $STRAIT_API_KEY" \
  -d '{"payload": {"order_id": "ord_123"}, "debug_mode": true}'
```

**After the run starts**: call the debug endpoint on an existing run.

```bash
curl -X POST https://api.strait.dev/v1/runs/{runID}/debug \
  -H "Authorization: Bearer $STRAIT_API_KEY" \
  -d '{"debug_mode": true}'
```

The second approach is useful when a run is already in progress and you want to start capturing data mid-execution.

## What's collected

| Data type | Description |
|---|---|
| Events | State changes and events emitted during the run |
| Checkpoints | Snapshots of run state at each step boundary |
| Outputs | Final and intermediate outputs from the run |
| Resource snapshots | CPU, memory, and other resource metrics during execution |

## Retrieving the bundle

Once the run finishes, fetch the full bundle:

```bash
curl https://api.strait.dev/v1/runs/{runID}/debug-bundle \
  -H "Authorization: Bearer $STRAIT_API_KEY"
```

```json
{
  "run_id": "run_abc123",
  "events": [
    {"type": "state_change", "from": "queued", "to": "executing", "at": "2026-01-15T10:00:01Z"},
    {"type": "tool_call", "name": "search", "duration_ms": 340, "at": "2026-01-15T10:00:02Z"}
  ],
  "checkpoints": [
    {"step": 1, "state": "executing", "at": "2026-01-15T10:00:01Z"}
  ],
  "outputs": [
    {"step": 2, "value": "Q4 revenue grew 12% year-over-year..."}
  ],
  "resource_snapshots": [
    {"at": "2026-01-15T10:00:02Z", "cpu_percent": 42.1, "memory_mb": 256}
  ]
}
```

## Performance note

Debug mode adds overhead: extra database writes for every event, checkpoint, and resource sample. Use it when investigating a specific failure, not on every run in production.
