Some workflows take longer than a single request: an order may wait for payment, a deployment may wait for approval, or a pipeline may pause until the next business day.

Durable workflows keep that progress in Strait until every step finishes or fails.

## When to Use Them

Use durable workflows when a process needs to wait:

- **Order fulfillment**: Create an order, wait for payment, reserve inventory, then send confirmation.
- **Human approval**: Run checks, pause for a reviewer, then continue.
- **Delayed follow-up**: Run a step, sleep for a period, then check the result later.

## Wait Types

Durable workflows support three wait types:

| Wait type | Use it when |
|---|---|
| `sleep` | The workflow should continue after a known duration. |
| `wait_for_event` | An external system will send a matching event later. |
| `approval` | A person must approve or reject the step. |

While a workflow waits, it does not keep a worker busy.

## Recovery Contract

Strait persists workflow progress at the step boundary. If a worker loses its heartbeat while a job step is running, Strait retries the step when attempts remain and marks it crashed only after attempts are exhausted.

Write job steps so they can run more than once safely. Use idempotency keys with downstream systems, store external operation IDs, and make repeated callbacks or writes converge on the same result.

## Example

```json
{
  "steps": [
    { "step_ref": "create-order", "job_id": "order-create" },
    {
      "step_ref": "wait-for-payment",
      "type": "wait_for_event",
      "event_key": "payment.confirmed.{{create-order.order_id}}",
      "timeout_secs": 86400,
      "depends_on": ["create-order"]
    },
    {
      "step_ref": "approve-high-value",
      "type": "approval",
      "timeout_secs": 3600,
      "depends_on": ["wait-for-payment"]
    },
    {
      "step_ref": "reserve-inventory",
      "job_id": "inventory-reserve",
      "depends_on": ["approve-high-value"]
    },
    {
      "step_ref": "send-confirmation",
      "job_id": "notifications-send",
      "depends_on": ["reserve-inventory"]
    }
  ]
}
```

This workflow waits up to 24 hours for payment, pauses for approval on high-value orders, reserves inventory, and sends a confirmation.

## Completion Estimates

Add `expected_duration_secs` to steps when you want Strait to estimate when a workflow should finish.

```json
{
  "steps": [
    { "step_ref": "validate", "expected_duration_secs": 5 },
    { "step_ref": "charge", "depends_on": ["validate"], "expected_duration_secs": 10 },
    { "step_ref": "notify", "depends_on": ["charge"], "expected_duration_secs": 3 }
  ]
}
```

In this example, the expected duration is 18 seconds. If the workflow has parallel branches, Strait uses the branch that takes the longest.

## Step Notifications

Steps can notify your team when they complete, fail, or are skipped.

```json
{
  "step_ref": "charge",
  "stage_notifications": {
    "on_complete": true,
    "on_failure": true,
    "on_skipped": true
  }
}
```

Use step notifications for approval handoffs, failed payment steps, or any stage where a person needs to know what happened.
