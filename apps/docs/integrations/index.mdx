Strait connects to PostgreSQL, Redis, Sequin, and optional monitoring tools.

## Core Infrastructure

<CardGroup cols={2}>
  <Card title="PostgreSQL" icon="database">
    Required. Strait stores all state in Postgres and uses `SELECT ... FOR UPDATE SKIP LOCKED` plus `LISTEN`/`NOTIFY` for queue semantics and worker wake-ups.
  </Card>
  <Card title="Redis" icon="bolt">
    Required. Powers rate limiting, pub/sub, SSE streaming for the dashboard, live run updates, and gRPC worker-plane signalling.
  </Card>
  <Card title="Sequin" icon="bolt">
    Required. Streams PostgreSQL changes through CDC so Strait can push real-time updates without polling the database.
  </Card>
</CardGroup>

## Monitoring

<CardGroup cols={2}>
  <Card title="OpenTelemetry" icon="signal">
    Export traces, metrics, and logs via OTLP to any OpenTelemetry-compatible backend.
  </Card>
  <Card title="Grafana" icon="chart-line" href="/operations/observability">
    Pre-built dashboards for job throughput, run latency, queue depth, and error rates. Shipped in `apps/strait/monitoring/grafana/`.
  </Card>
  <Card title="ClickHouse" icon="chart-bar" href="/concepts/clickhouse-analytics">
    Analytics for run history, cost, and throughput.
  </Card>
  <Card title="Sentry" icon="bug">
    Error reporting with run context, job metadata, and workflow step information.
  </Card>
</CardGroup>
