Strait connects to PostgreSQL, Redis, and optional monitoring tools.

## Core Infrastructure

<CardGroup cols={2}>
  <Card title="PostgreSQL" icon="database">
    The only required dependency. Strait uses database locks for queue semantics, LISTEN/NOTIFY for real-time updates, and stores all state in Postgres.
  </Card>
  <Card title="Redis" icon="bolt">
    Required. Powers real-time updates, pub/sub notifications, and live dashboard updates.
  </Card>
</CardGroup>

## Observability

<CardGroup cols={2}>
  <Card title="OpenTelemetry" icon="signal">
    Export traces, metrics, and logs via OTLP to any OpenTelemetry-compatible backend.
  </Card>
  <Card title="Grafana Cloud" icon="chart-line">
    Pre-built dashboards for job throughput, run latency, queue depth, and error rates.
  </Card>
  <Card title="ClickHouse" icon="chart-bar" href="/concepts/clickhouse-analytics">
    Analytics for run history, cost, and throughput.
  </Card>
  <Card title="Sentry" icon="bug">
    Automatic error reporting with run context, job metadata, and workflow step information.
  </Card>
</CardGroup>

## Log Forwarding

<CardGroup cols={2}>
  <Card title="Datadog" icon="chart-area" href="/concepts/log-drains">
    Forward job execution logs to Datadog in real time via log drains.
  </Card>
  <Card title="Splunk" icon="magnifying-glass" href="/concepts/log-drains">
    Stream structured logs to Splunk for centralized analysis.
  </Card>
</CardGroup>
