This guide shows how to write jobs that run inside Strait using the official SDKs. Your job code receives payloads, reports progress, and returns results.

<Info>
  For per-language API documentation, see the [SDK Reference](/sdks/overview).
</Info>

<Info>
Managed agents use a separate runtime package, [`@strait/agents`](/docs/sdks/agents), for run-token callbacks, workflow state, and provider adapters.
</Info>

## Writing a Job Handler

When Strait triggers a job, it calls your HTTP endpoint with the run payload and an SDK token. Here is what happens inside your handler:

```typescript
// Your Express/Fastify/Hono endpoint that Strait calls
app.post('/jobs/send-email', async (req, res) => {
  const { payload, run_id, attempt } = req.body;
  const token = req.headers['x-sdk-token'];

  // Create SDK client for this run
  const ctx = createRunContext({ token });

  // Report progress
  await ctx.log('Sending email to ' + payload.to);
  await ctx.heartbeat();

  // Do the work
  const result = await sendEmail(payload);

  // Return result
  res.json({ sent: true, messageId: result.id });
});
```

Strait delivers the request with:

- **`req.body.payload`** -- the data you passed when triggering the job.
- **`req.body.run_id`** -- unique identifier for this run.
- **`req.body.attempt`** -- which attempt this is (starts at 1, increments on retry).
- **`x-sdk-token` header** -- a short-lived JWT for calling SDK endpoints during execution.

## SDK Run Context

The run context gives your job handler methods to communicate state back to Strait:

| Method | Purpose |
|---|---|
| `log(message, level?)` | Send structured logs visible in the dashboard |
| `heartbeat()` | Signal the job is alive (call every 30s in long-running jobs) |
| `checkpoint(state)` | Save resumable state; on retry, your handler receives this state |
| `reportUsage(provider, model, tokens, cost)` | Track AI model usage and costs |
| `spawnChild(jobName, payload)` | Create a child job run |
| `progress(percent, message?)` | Report completion percentage |
| `annotate(key, value)` | Attach metadata (max 50 annotations per run) |
| `output(data)` | Store structured results or artifacts |
| `waitForEvent(eventKey, timeoutSecs)` | Pause until an external event arrives |

### Example: Long-Running Job with Checkpoints

```typescript
app.post('/jobs/process-csv', async (req, res) => {
  const { payload, run_id } = req.body;
  const ctx = createRunContext({ token: req.headers['x-sdk-token'] });

  const rows = await fetchRows(payload.fileUrl);
  let processed = payload.checkpoint?.processed ?? 0;

  for (let i = processed; i < rows.length; i++) {
    await processRow(rows[i]);
    processed = i + 1;

    // Checkpoint every 100 rows so retries resume here
    if (processed % 100 === 0) {
      await ctx.checkpoint({ processed });
      await ctx.progress(Math.round((processed / rows.length) * 100));
      await ctx.heartbeat();
    }
  }

  res.json({ totalProcessed: processed });
});
```

## Authoring DSL

For TypeScript, the SDK provides `defineJob` and `defineWorkflow` helpers that declare jobs with typed payloads, retry policies, and timeout configuration:

```typescript
import { defineJob } from '@strait/ts';

export const sendEmail = defineJob({
  name: 'send-email',
  retry: { maxAttempts: 3, backoff: 'exponential' },
  timeout: '30s',
  handler: async (ctx, payload: { to: string; subject: string }) => {
    await ctx.log(`Sending to ${payload.to}`);
    return await mailer.send(payload);
  },
});
```

See [TypeScript SDK](/sdks/typescript) and [Python SDK](/sdks/python) for language-specific DSL documentation.

## Testing

Use `createTestContext` to unit test job handlers without connecting to Strait:

```typescript
import { createTestContext } from '@strait/ts/testing';

test('send-email handler', async () => {
  const ctx = createTestContext();

  const result = await sendEmail.handler(ctx, {
    to: 'alice@example.com',
    subject: 'Hello',
  });

  expect(result.messageId).toBeDefined();
  expect(ctx.logs).toContainEqual(
    expect.objectContaining({ message: expect.stringContaining('alice') })
  );
});
```

The test context captures logs, checkpoints, and heartbeats so you can assert on them without running the full system.

### Wait for External Events

Pause a run and wait for an external event (approvals, webhooks, third-party callbacks):

<CodeGroup>
  ```ts TypeScript
  // Pause the run
  await runClient.waitForEvent({
    pathParams: { runID: "run-123" },
    body: {
      event_key: "approval:order-789",
      timeout_secs: 7200,
    },
  });

  // Later, send the event (using an API key client)
  const apiClient = createClient({
    baseUrl: "https://api.strait.dev",
    auth: { type: "bearer", token: process.env.STRAIT_API_KEY! },
  });

  await apiClient.sendEvent({
    pathParams: { eventKey: "approval:order-789" },
    body: { payload: { approved: true } },
  });
  ```

  ```python Python
  # Pause the run
  run_client.sdk_runs.wait_for_event("run-123", {
      "event_key": "approval:order-789",
      "timeout_secs": 7200,
  })

  # Later, send the event
  api_client = Client.from_file()
  api_client.event_triggers.send("approval:order-789", {
      "payload": {"approved": True},
  })
  ```
</CodeGroup>

<Tip>
Use the `heartbeat` endpoint regularly (every 30 seconds) in long-running jobs to prevent timeout.
</Tip>

## Managed Agent Runtime Integration

Managed agents in this repository do not use the public management client inside the executing runtime. They use `@strait/agents`, which is built specifically for `/sdk/v1/runs/{runID}/...` callbacks.

Typical usage inside a running agent:

```ts
import { StraitContext } from "@strait/agents";

const ctx = StraitContext.fromEnv();

await ctx.log({ message: "agent booted" });
await ctx.progress({ percent: 10, message: "loading tools" });
await ctx.reportUsage({
  provider: "openai",
  model: "gpt-5.4-mini",
  promptTokens: 900,
  completionTokens: 240,
});
await ctx.complete({ status: "ok" });
```

For the full runtime contract, see [Agents SDK](/docs/sdks/agents) and [Local Agent Development](/docs/guides/local-agent-development).
