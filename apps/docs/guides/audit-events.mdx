Audit events record every change in your Strait project. Query them to investigate incidents, meet compliance requirements, or build activity feeds. In Strait Cloud, audit log access, export, and chain verification require Scale or above. Self-hosted deployments have no billing gate.

## What Gets Recorded

Strait logs every change, organized by category:

- **Job operations**: creation, updates, deletion, trigger events
- **Run operations**: state transitions, retries, cancellations
- **Workflow operations**: creation, updates, approval actions, step transitions
- **Auth operations**: API key creation, rotation, revocation, OIDC token exchanges
- **Settings changes**: project configuration, retention policy updates, quota changes
- **Webhook operations**: subscription creation, deletion, delivery replays, endpoint tests
- **Worker operations**: worker connections, force-disconnect requests, and worker-mode task routing

Each event includes the actor, timestamp, resource type, resource ID, and a description of the change.

## Querying Events

List audit events with `GET /v1/audit-events`. Requires the `rbac.manage` permission and, in Strait Cloud, the Scale plan or above.

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://api.strait.dev/v1/audit-events?limit=25"
```

### Filtering

All filters combine with AND logic. Every filter is optional.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `actor_id` | string | Who performed the action |
| `resource_type` | string | Resource kind (`job`, `run`, `api_key`, etc.) |
| `resource_id` | string | Specific resource identifier |
| `from` | string | Start of time window (RFC3339) |
| `to` | string | End of time window (RFC3339) |

**Find all actions by a specific user:**

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://api.strait.dev/v1/audit-events?actor_id=user_abc123"
```

**Find all job changes in a specific week:**

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://api.strait.dev/v1/audit-events?resource_type=job&from=2026-06-01T00:00:00Z&to=2026-06-08T00:00:00Z"
```

**Track the full history of a single run:**

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://api.strait.dev/v1/audit-events?resource_type=run&resource_id=run_abc123&order=asc"
```

<Warning>
The maximum time window is 90 days. Use pagination with narrower windows for longer lookback periods.
</Warning>

### Pagination

Results are cursor-based. The response includes `next_cursor` when more pages exist. Pass it as the `cursor` query parameter.

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://api.strait.dev/v1/audit-events?limit=25&cursor=2026-06-15T10:30:00.000000000Z"
```

## Exporting

Export audit events with `GET /v1/audit-events/export`. Both `from` and `to` are required (maximum 90-day window). The response is streamed. In Strait Cloud, export uses the same Scale+ audit log gate as audit queries.

| Format | Content-Type | Description |
| :--- | :--- | :--- |
| `json` | `application/json` | JSON array (default) |
| `ndjson` | `application/x-ndjson` | Newline-delimited JSON |
| `csv` | `text/csv` | CSV with header row |

```bash
curl -H "Authorization: Bearer $TOKEN" \
  -o audit-events.json \
  "https://api.strait.dev/v1/audit-events/export?from=2026-06-01T00:00:00Z&to=2026-06-30T00:00:00Z&format=ndjson"
```

Exports support the same `actor_id` and `resource_type` filters as the list endpoint. Large exports should use narrow time windows.

## Chain Verification

Verify the audit log when you need evidence that history has not been changed:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://api.strait.dev/v1/audit-events/verify"
```

For large audit logs, use incremental mode to check only events added since the last verification:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://api.strait.dev/v1/audit-events/verify?incremental=true"
```

<Tip>
Run full verification periodically (e.g. weekly) and incremental checks more frequently.
</Tip>

## Key Rotation

Rotate the audit signing key with `POST /v1/projects/{id}/audit/rotate-key`. Existing audit history remains verifiable after rotation.

## Retention

Audit events are pruned automatically after 365 days by default (`AUDIT_RETENTION_DEFAULT_DAYS`). Per-project overrides are available:

```bash
curl -X PUT \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"days": 730}' \
  "https://api.strait.dev/v1/projects/proj_abc123/audit/retention"
```

Set `days` to `0` to disable retention trimming entirely.
