Audit events record every change in your Strait project. Query them to investigate incidents, meet compliance requirements, or build activity feeds.

## What Gets Recorded

Strait logs every change, organized by category:

- **Job operations**: creation, updates, deletion, trigger events
- **Run operations**: state transitions, retries, cancellations
- **Workflow operations**: creation, updates, approval actions, step transitions
- **Auth operations**: API key creation, rotation, revocation, OIDC token exchanges
- **Settings changes**: project configuration, retention policy updates, quota changes
- **Webhook operations**: subscription creation, deletion, delivery replays, endpoint tests

Each event includes the actor, timestamp, resource type, resource ID, and a description of the change.

## Querying Events

List audit events with `GET /v1/audit-events`. Requires the `rbac.manage` permission.

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://strait.dev/v1/audit-events?limit=25"
```

### Filtering

All filters combine with AND logic. Every filter is optional.

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `actor_id` | string | Who performed the action |
| `resource_type` | string | Resource kind (`job`, `run`, `api_key`, etc.) |
| `resource_id` | string | Specific resource identifier |
| `from` | string | Start of time window (RFC3339) |
| `to` | string | End of time window (RFC3339) |

**Find all actions by a specific user:**

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://strait.dev/v1/audit-events?actor_id=usr_abc123"
```

**Find all job changes in a specific week:**

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://strait.dev/v1/audit-events?resource_type=job&from=2025-06-01T00:00:00Z&to=2025-06-08T00:00:00Z"
```

**Track the full history of a single run:**

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://strait.dev/v1/audit-events?resource_type=run&resource_id=run_xyz789&order=asc"
```

<Warning>
The maximum time window is 90 days. Use pagination with narrower windows for longer lookback periods.
</Warning>

### Pagination

Results are cursor-based. The response includes `next_cursor` when more pages exist. Pass it as the `cursor` query parameter.

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://strait.dev/v1/audit-events?limit=25&cursor=2025-06-15T10:30:00.000000000Z"
```

## Exporting

Export audit events with `GET /v1/audit-events/export`. Both `from` and `to` are required (maximum 90-day window). The response is streamed.

| Format | Content-Type | Description |
| :--- | :--- | :--- |
| `json` | `application/json` | JSON array (default) |
| `ndjson` | `application/x-ndjson` | Newline-delimited JSON |
| `csv` | `text/csv` | CSV with header row |

```bash
curl -H "Authorization: Bearer $TOKEN" \
  -o audit-events.json \
  "https://strait.dev/v1/audit-events/export?from=2025-06-01T00:00:00Z&to=2025-06-30T00:00:00Z&format=ndjson"
```

Exports support the same `actor_id` and `resource_type` filters as the list endpoint. Exports are capped at 1,000,000 rows by default and rate-limited to 10 per project per hour.

## Chain Verification

Every audit event is HMAC-SHA256 signed and chained to its predecessor. Verify the chain is intact:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://strait.dev/v1/audit-events/verify"
```

For large audit logs, use incremental mode to check only events added since the last verification:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "https://strait.dev/v1/audit-events/verify?incremental=true"
```

<Tip>
Run full verification periodically (e.g. weekly) and incremental checks more frequently.
</Tip>

## Key Rotation

Rotate the HMAC signing key with `POST /v1/projects/{id}/audit/rotate-key`. The rotation is atomic -- old signatures remain verifiable because the chain verifier detects key epoch boundaries automatically.

## Retention

Audit events are pruned automatically after 365 days by default (`AUDIT_RETENTION_DEFAULT_DAYS`). Per-project overrides are available:

```bash
curl -X PUT \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"days": 730}' \
  "https://strait.dev/v1/projects/proj_abc/audit/retention"
```

Set `days` to `0` to disable retention trimming entirely. When events are deleted, a tombstone anchor preserves chain continuity so verification still passes.

## Failed Audit Events

If an audit event cannot be saved after retries, Strait keeps it separate so an administrator can inspect and replay it.

| Endpoint | Method | Description |
| :--- | :--- | :--- |
| `/v1/audit/deadletter` | GET | List failed audit rows |
| `/v1/audit/deadletter/{id}` | GET | Inspect one failed audit row |
| `/v1/audit/deadletter/{id}/replay` | POST | Replay an event into the primary chain |
