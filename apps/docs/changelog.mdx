## 2026-04

### Strait Notify Backend and Operations Documentation

- Added Strait Notify backend foundation for subscribers, topics, templates, categories, preferences, inbox items, messages, providers, unsubscribe tokens, dedup keys, digest batches, and escalation states.
- Added Notify trigger and management API surface, including template preview/test flows and delivery listing.
- Added scheduler processing for due scheduled messages, digest batch flushes, and approval-reminder escalation tiers.
- Added escalation lifecycle endpoints by step run:
  - `GET /v1/notify/escalations/step-runs/{stepRunID}`
  - `POST /v1/notify/escalations/step-runs/{stepRunID}/acknowledge`
  - `POST /v1/notify/escalations/step-runs/{stepRunID}/complete`
- Synced OpenAPI registration for new escalation endpoints.
- Expanded docs for digest and escalation behavior:
  - API Reference overview updates
  - Workflow approval guide escalation section
  - New operations runbook: `/operations/notify-digest-escalation-runbook`
- Hardened notify subscriber auth and unsubscribe flows:
  - subscriber tokens now enforce issuer/audience claims
  - unsubscribe tokens use cryptographically secure random generation with hashed token lookup at rest
- Added notify delivery reliability controls:
  - scheduled message retries with capped exponential backoff
  - provider fallback chains for email delivery (`fallback_id`)
  - inbox projection idempotency via `inbox_items.message_id` uniqueness
- Added configurable notify scheduler knobs (`NOTIFY_*`) for retries, digest payload limits, and escalation tier pacing.
- Added notify-focused Grafana dashboard asset and tuned notify alert thresholds in `packages/monitoring/alerts-strait-core.yaml`.

## 2026-03

### Workflow Versioning and Breaking Change Detection

- You can now flag a workflow update as a breaking change. When active runs exist on the previous version, Strait records a `workflow.updated_breaking` audit event and returns details about in-flight runs in the response.
- New `GET /v1/workflows/{workflowID}/active-versions` endpoint lists all versions with active runs, including per-status breakdowns (pending, running, paused).

### Authentication, Authorization, and Security

- Added optional OIDC bearer-token authentication for user traffic.
- Expanded RBAC with role inheritance and optional lineage introspection.
- Added resource policies and tag policies for fine-grained authorization control.
- Added bulk member assignment via `POST /v1/members/bulk`.
- Added audit event API with filtering and ordering (`GET /v1/audit-events`).
- Added API key rotation with configurable grace windows (`POST /v1/api-keys/{keyID}/rotate`), with matching CLI support (`strait api-keys rotate`).
- Webhooks now include `X-Strait-Timestamp` and `X-Strait-Signature` headers for verifying authenticity.

### Performance and Analytics

- New `GET /v1/analytics/performance` endpoint provides slowest jobs, throughput, and health summary aggregations.
- Significant performance improvements across bulk operations, webhook retries, and query execution.

### Batch Operations

- Track bulk trigger requests with the new batch operations API. Query batch status via `GET /v1/batch-operations`.

### Log Drains

- Stream logs to external systems via configurable HTTP drain endpoints. Full CRUD at `/v1/log-drains`.

### Event Sources

- Manage inbound events with subscriptions and filter expressions. Dispatch events to jobs or workflows via `POST /v1/events/dispatch`.

### Concurrency and Rate Limiting

- New `max_concurrency_per_key` setting on jobs and `concurrency_key` on runs for per-tenant or per-resource throttling.
- Named rate limit keys allow multiple rate limits per job, extending the existing flat rate limit configuration.

### Run Improvements

- Added `default_run_metadata` on jobs, automatically applied to every new run.
- Filter run listings by payload content with the `payload_contains` query parameter.
- Runs now include `batch_id`, `concurrency_key`, `created_by`, `tags`, and `job_version_id` fields.
- New endpoints for run checkpoints, usage, tool calls, and outputs.

### Bulk Run Operations

- Added `POST /v1/runs/bulk-cancel`, `POST /v1/runs/bulk-replay`, `POST /v1/runs/bulk-dlq-replay`, and `POST /v1/runs/bulk-cancel-all`.
- Workflow runs support bulk cancel, individual retry, and step retry.

### Documentation

- Expanded OpenAPI specification.
- Added operations guides for monitoring, rollouts, and authorization incident response.
