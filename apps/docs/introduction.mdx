## What is Strait

Strait runs your background jobs and orchestrates multi-step workflows. You define the jobs; it handles queuing, retries, scheduling, dependencies, and monitoring.

One Go binary. PostgreSQL for state and the queue. Redis for live updates. No RabbitMQ, no SQS, no Kafka.

## Why teams switch

Most teams start with a queue and a retry loop. Then they need scheduling. Then dependencies between jobs. Then approval gates, then cost limits on expensive API calls. Six months later they're maintaining five systems that don't talk to each other.

Strait collapses that into one service:

- Every run has a clear lifecycle -- `queued` to `executing` to `completed` or `failed`. Retries use exponential, linear, fixed, or custom delays with jitter. Runs that exhaust their retries land in `dead_letter` for inspection and replay.
- Workflows handle dependencies, conditional branches, approval gates, sub-workflows, and durable event waits. The graph is validated before it runs.
- Live observability is built in: where each run is, why it failed, how long it took, what it cost. Streamed over SSE, not polled.
- Five SDKs with full feature parity: TypeScript, Python, Go, Ruby, Rust.
- Self-host with PostgreSQL and Redis, or use the hosted version at [app.strait.dev](https://app.strait.dev). Same API either way.

## What you can build with it

- **Background jobs** — scheduled imports, report generation, cache warming, cleanup, recurring maintenance.
- **Webhook consumers** -- durable processing of events from external services with retries and dead-letter recovery for anything that exhausts its attempts.
- **Cost-aware AI pipelines** — multi-step workflows with per-run and per-project spend limits, approval gates, and conditional branches. Tracks cost in micro-USD.
- **Bulk operations** — trigger jobs in batches with priority ordering and idempotency deduplication.
- **ETL** — DAG workflows with parallel steps and template-driven data flow between stages.
- **Cron** — five-field expressions with timezone support and execution windows.
