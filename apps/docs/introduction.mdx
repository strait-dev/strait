## What is Strait?

Strait is an open-source platform that runs your background jobs and orchestrates multi-step workflows. You define what needs to happen. Strait handles the retries, scheduling, dependencies, and monitoring.

One Go binary. PostgreSQL for state and queuing. Redis for real-time updates. No RabbitMQ. No SQS. Just deploy and start running jobs.

## Two execution products, one platform

Strait serves two different workload shapes:

- **Jobs** run durable background work and workflows on Fly Machines
- **Agents** run LLM-centric orchestration and tool loops on Cloudflare Workers

They share the same control plane, dashboard, authentication, telemetry, and billing account, but they are optimized for different execution models.

Use Jobs for container-backed, long-running, or compute-heavy work.

Use Agents for low-latency model orchestration, tool calling, streaming, and budgeted LLM execution.

When an agent needs heavy compute, the recommended pattern is to trigger a Job or Workflow rather than forcing that work through the agent runtime.

## Why teams switch to Strait

Most teams start with a simple queue and a retry loop. Then they need scheduling. Then workflow dependencies. Then approval gates. Then cost tracking for expensive API calls. Before long, they're maintaining five different systems that don't talk to each other.

Strait replaces that patchwork with one system:

- **Jobs fail predictably.** Every run follows a clear lifecycle from queued to completed (or failed). Retries use exponential backoff, fixed delays, or custom sequences. Exhausted runs go to a review queue so you can inspect and replay them.
- **Workflows handle complex dependencies.** Define step dependencies, approval gates, sub-workflows, event waits, and sleep delays. Strait validates the graph before running it.
- **Everything is observable.** You see exactly where every run is, why it failed, how long it took, and what it cost. Real-time streaming, not polling.
- **Five SDKs, same architecture.** TypeScript, Python, Go, Ruby, and Rust. Pick your language, define your jobs, run them anywhere.
- **Self-host or use managed.** Deploy on your infrastructure with just PostgreSQL, or use the managed platform at [app.strait.dev](https://app.strait.dev).

## Use Cases

Strait fits these patterns:

**Background Jobs**: Scheduled data imports, report generation, cache warming, cleanup tasks, and recurring maintenance operations.

**Webhook Consumers**: Process events from external services with retries and delivery guarantees. Failed deliveries go to a review queue so nothing is silently lost.

**Cost-Tracked Pipelines**: Multi-step pipelines with human approval gates, conditional execution, cost tracking per run and per project, and sub-workflow nesting. Use for any workload where you need to control spend.

**Agent Orchestration + Heavy Work Handoff**: Let a Cloudflare-backed agent reason, stream, and select tools quickly, then escalate heavier work to Fly-backed Jobs or Workflows.

**Batch Processing**: Bulk job triggering with configurable batch sizes, priority ordering, and idempotency deduplication.

**Data Pipelines**: ETL workflows with parallel steps, transform stages, and aggregation.

**Cron Jobs**: Standard 5-field cron expressions with timezone support and execution windows.
