## What Strait Does

Strait runs background jobs and multi-step workflows. You define the work, where it runs, and when it should happen. Strait handles the queue, retries, schedules, dependencies, and run history.

It ships as a single Go binary backed by PostgreSQL for state and queued work, Redis for the transport plane and rate limiting, and Sequin for CDC-driven updates. There is no separate message broker or workflow engine to run.

## When to Use It

Strait is useful when a simple queue is no longer enough:

- You need retries, timeouts, and `dead_letter` handling.
- You need recurring jobs with clear run history.
- You need workflows where one step waits for another.
- You need approvals, event waits, or long-running processes.
- You want to self-host the control plane and keep job code on your own infrastructure.

Each run moves through clear states: `queued`, `executing`, then `completed`, `failed`, or another terminal state. Failed runs retry according to the job policy. If retries are exhausted, the run moves to `dead_letter` so it can be inspected and replayed.

## Common Uses

- **Background jobs**: imports, reports, cache warming, cleanup, and recurring maintenance.
- **Webhook consumers**: process external events with retries and replay for failed work.
- **Data pipelines**: run steps in order, pass results between steps, and retry only what failed.
- **Approval workflows**: route high-risk steps through explicit review before continuing.
- **Scheduled work**: use cron expressions, time zones, and execution windows.
