This report records the May 25, 2026 local validation pass for the 0.1.6 performance and reliability work. The host was an Apple M1 Pro. Integration tests used local Docker testcontainers when available.

## Scope

The 0.1.6 work focused on lock contention, queue bloat, request-path query count, workflow recovery, CDC redelivery, and runtime tuning. The code now defaults to the faster and safer paths instead of keeping legacy feature flags for the main behavior.

## Before and after

| Area | Before | After | Result |
|---|---:|---:|---|
| Workflow trigger enqueue with empty idempotency key | blocking advisory lock per same-job enqueue | no idempotency lock unless key is non-empty | removes lock-timeout failure mode |
| Stalled workflow default action | `log_only` | `reconcile` | stalled runs auto-resume by default |
| Trigger admission | blocking advisory lock on hot trigger path | row-atomic admission and retryable contention | removes per-job lock serialization |
| Health stats hot path | repeated history `COUNT` scans | incremental counters and pass-through reads | O(1) stats on hot paths |
| Execution trace persistence | write JSON trace by default | `EXECUTION_TRACE_MODE=off` by default | removes default trace update cost |
| Step visibility race | missing step row could system-fail run | transient requeue path | avoids spurious `SystemFailed` |
| Heartbeat loss | terminal `Crashed` immediately | retry with backoff while attempts remain | crash recovery follows retry policy |
| CDC projection publish failure | NACK and redeliver all side effects | ACK projection-only messages, NACK durable handler failures | removes fan-out redelivery amplification |
| Workflow-step CDC channel | published unused `cdc:workflow_run:%s:steps` messages | not subscribed or registered | removes wasted Redis publish work |
| Adaptive idle behavior | could shed workers at zero utilization | preserves warm capacity at cold idle | improves burst pickup after idle |
| Pool tuner pressure signal | raw wait-count increase meant pressure | pressure requires full pool or acquire-wait evidence | avoids false pool pressure advice |

Acceptance gates:

- no lock-timeout HTTP 500s on trigger or workflow trigger contention.
- exact trigger admission quotas under concurrency.
- no duplicate queue claims, lost runs, duplicate webhook deliveries, or dropped audit intents.
- `GetJobHealthStats` does not scan all job history on request hot paths.
- `execution_trace` update cost is near zero unless trace persistence is explicitly enabled.
- CDC fan-out failures do not redeliver durable side effects.
- PgQue queue and outbox claim-log bloat stay flat or improve against the baseline report.
