This report summarizes the local benchmark pass for the PgQue-inspired queue work. The latest PgQue gate was captured on June 2, 2026 on an Apple M1 Pro with testcontainer Postgres.

## Result

Switch the queue engine to `pgque` by default. Keep the outbox and workflow progression engines on `batchlog`. PgQue is used only as Strait's ready-event log; Strait still owns orchestration state, run state, retries, concurrency, workers, observability, and APIs.

The latest 1k-run gate keeps duplicate and lost claims at zero, reduces WAL by about 4.98 MB versus the legacy queue path, and removes `queue_entries` dead tuple growth from the PgQue queue path. `job_run_active_claims` remains append-only with zero dead tuples; its live table and index growth are bounded by the bloat gate.

Keep `QUEUE_USE_DENORMALIZED_DEQUEUE=true` for the legacy rollback path. Continue using batchlog for outbox and workflow progression because those systems still use separate claim/event storage from the queue engine.

The latest claim-only ack pass makes batchlog and legacy latency overlap locally while reducing successful-promotion hot table mutation.

## Reliability Checks

Passed targeted checks covered:

- no duplicate claims under concurrent batchlog queue workers.
- batchlog lease expiry redelivery before worker start.
- direct `queued -> executing` transition for batchlog queue starts.
- batchlog lease read model blocking additional claims while a run is leased but not yet executing.
- outbox batchlog retryable release, terminal quarantine, concurrent flusher ownership, write-time claims, empty-batch avoidance, and async promoted-row archive.
- workflow progression event claim, dedupe, ack, and fuzzed grouping behavior.

## Rollout

Recommended flags:

```bash
QUEUE_USE_DENORMALIZED_DEQUEUE=true
QUEUE_ENGINE=pgque
OUTBOX_ENGINE=batchlog
WORKFLOW_PROGRESSION_ENGINE=batchlog
```

Legacy rollback settings:

```bash
QUEUE_ENGINE=legacy
OUTBOX_ENGINE=legacy
WORKFLOW_PROGRESSION_ENGINE=legacy
```

Rollback is immediate: set the engine variables above to `legacy` and restart workers/API processes. The batchlog tables can remain in place while the legacy paths run.
