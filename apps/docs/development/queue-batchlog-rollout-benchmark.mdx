This report summarizes the local benchmark pass for the PgQue-inspired queue work. The latest run was captured on May 24, 2026 on an Apple M1 Pro with testcontainer Postgres.

## Result

Switch the queue, outbox, and workflow progression engines to `batchlog` by default. The optimized batchlog queue beats the legacy queue in the comparative bloat gate while reducing `job_runs` table and index growth. The outbox path now avoids mutating `enqueue_outbox.consumed_at` for successful batchlog promotion; successful rows are acked through `outbox_claims` and archived asynchronously.

Keep `QUEUE_USE_DENORMALIZED_DEQUEUE=true` for the legacy rollback path. Continue optimizing batchlog outbox, but use it as the default because successful promotions no longer update the hot outbox ledger row.

The queue batchlog path improved from 483.45 ms/op to 38.35 ms/op in the local benchmark, mainly by moving backfill, sealing, and lease reclamation out of `DequeueN` and maintaining queue entries at write time. The outbox batchlog path improved from 397.63 ms/op after write-time claims to 282.15 ms/op after removing immediate archive from the flusher transaction. The latest claim-only ack pass makes batchlog and legacy latency overlap locally while reducing successful-promotion hot table mutation.

## Reliability Checks

Passed targeted checks covered:

- no duplicate claims under concurrent batchlog queue workers.
- batchlog lease expiry redelivery before worker start.
- direct `queued -> executing` transition for batchlog queue starts.
- batchlog lease read model blocking additional claims while a run is leased but not yet executing.
- outbox batchlog retryable release, terminal quarantine, concurrent flusher ownership, write-time claims, empty-batch avoidance, and async promoted-row archive.
- workflow progression event claim, dedupe, ack, and fuzzed grouping behavior.

## Rollout

Recommended flags:

```bash
QUEUE_USE_DENORMALIZED_DEQUEUE=true
QUEUE_ENGINE=batchlog
OUTBOX_ENGINE=batchlog
WORKFLOW_PROGRESSION_ENGINE=batchlog
```

Legacy rollback settings:

```bash
QUEUE_ENGINE=legacy
OUTBOX_ENGINE=legacy
WORKFLOW_PROGRESSION_ENGINE=legacy
```

Rollback is immediate: set the engine variables above to `legacy` and restart workers/API processes. The batchlog tables can remain in place while the legacy paths run.
