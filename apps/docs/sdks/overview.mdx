Strait provides verified package installs for TypeScript, Python, and Go. Ruby and Rust source repositories exist, but their package-registry installs are not documented as production-ready until they are published and verified.

Use a verified SDK to create jobs, trigger runs, inspect status, define workflows, and report progress from job handlers.

## Install

| Language | Install |
|---|---|
| TypeScript | `npm install @strait/ts` |
| Python | `pip install strait-python` |
| Go | `go get github.com/strait-dev/strait-go` |
| Ruby | Source preview: [strait-dev/strait-ruby](https://github.com/strait-dev/strait-ruby) |
| Rust | Source preview: [strait-dev/strait-rust](https://github.com/strait-dev/strait-rust) |

## Configure

Set your API key:

```bash
export STRAIT_API_KEY="strait_..."
```

Then create a client from the environment.

<CodeGroup>
  ```ts TypeScript
  import { createClientFromEnv } from "@strait/ts/node";

  const client = createClientFromEnv();
  ```

  ```python Python
  from strait import Client

  client = Client.from_env()
  ```

  ```go Go
  client, err := strait.NewClientFromEnv()
  if err != nil {
      return err
  }
  ```
</CodeGroup>

## Trigger a Job

<CodeGroup>
  ```ts TypeScript
  const run = await client.triggerJob({
    pathParams: { jobID: "send-email" },
    body: {
      payload: {
        to: "customer@example.com",
        template: "welcome",
      },
    },
  });

  console.log(run.id, run.status);
  ```

  ```python Python
  run = client.jobs.trigger(
      "send-email",
      {"payload": {"to": "customer@example.com", "template": "welcome"}},
  )

  print(run["id"], run["status"])
  ```

  ```go Go
  run, err := client.Jobs().Trigger(ctx, "send-email", strait.TriggerInput{
      Payload: map[string]any{
          "to": "customer@example.com",
          "template": "welcome",
      },
  })
  if err != nil {
      return err
  }

  fmt.Println(run.ID, run.Status)
  ```
</CodeGroup>

## Report Progress from a Job

When Strait calls your job endpoint, it includes a short-lived run token. Use that token to log progress, save checkpoints, and send heartbeats during long work.

```ts
app.post("/jobs/process-file", async (req, res) => {
  const ctx = createRunContext({
    token: req.headers["x-sdk-token"],
  });

  await ctx.log("Started file processing");
  await ctx.heartbeat();

  const result = await processFile(req.body.payload.file_url);

  res.json(result);
});
```

See [SDK Integration](/guides/sdk-integration) for a full handler example.

SDK run-token callbacks are authenticated with JWTs whose issuer is exactly `strait:run-token`. SDKs should reject callback tokens with any other issuer before using them for run-scoped API calls.
