The orchestrator is a production-grade Go job orchestrator inspired by Trigger.dev. It accepts job definitions via REST API, queues runs in Postgres using `SELECT FOR UPDATE SKIP LOCKED` (no external broker), dispatches them via HTTP to user endpoints, and handles retries with smart strategies. It ships as a single Go binary.

## Key Capabilities

<Info>
  The orchestrator is designed to be a single-binary solution that minimizes operational complexity while providing advanced features.
</Info>

- **13-state FSM**: Robust lifecycle management for every job run.
- **Workflow DAGs**: Complex orchestration with approval gates and output transforms.
- **Smart Retry**: Advanced backoff strategies with a 1-hour cap.
- **Cost Budgets**: Integrated AI cost tracking and budget enforcement.
- **CDC**: Real-time Postgres WAL change capture via Sequin Stream.
- **SDK Endpoints**: Specialized endpoints for logging, heartbeats, and progress.

## Tech Stack

The orchestrator is built with a modern, high-performance tech stack:

- **Go 1.26**: Single binary, no runtime dependencies.
- **PostgreSQL 18**: Primary store, job queue (SKIP LOCKED), and workflow state.
- **Redis 8**: Pub/Sub for SSE streaming and CDC event publishing.
- **Sequin**: CDC (Change Data Capture) from Postgres WAL.
- **OpenTelemetry**: Distributed tracing and Prometheus metrics.

## Project Structure

The core logic resides in the `internal/` directory:

- `internal/api/`: HTTP handlers, middleware, and auth.
- `internal/worker/`: Executor, pool, and backoff strategies.
- `internal/workflow/`: DAG validation, engine, and templates.
- `internal/queue/`: Queue interface + Postgres SKIP LOCKED.
- `internal/store/`: Database queries (pgx, raw SQL).
- `internal/domain/`: Core types and FSM definitions.
