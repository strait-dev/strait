---
title: "Feature Flags"
description: "Complete registry of feature flags available in Strait."
---

Strait uses feature flags (FF_* env vars) to control feature availability and system behavior. These flags allow you to toggle specific functionality without changing the core application configuration.

## Resilience and Retry

| Variable | Default | Description |
| :--- | :--- | :--- |
| `FF_SMART_RETRY` | `true` | Enables intelligent exponential backoff with jitter for job retries. |
| `FF_CIRCUIT_BREAKER` | `true` | Prevents cascading failures by tripping when downstream services are unhealthy. |
| `FF_BULKHEADS` | `true` | Isolates job execution resources to prevent a single job type from consuming all capacity. |
| `FF_RUN_DLQ` | `true` | Automatically moves permanently failed runs to a dead letter queue for manual inspection. |

## Data and Metadata

| Variable | Default | Description |
| :--- | :--- | :--- |
| `FF_PAYLOAD_VALIDATION` | `true` | Enforces JSON schema validation for job input and output payloads. |
| `FF_JOB_TAGS` | `true` | Enables key-value tagging for jobs and workflows to support advanced filtering. |
| `FF_RUN_ANNOTATIONS` | `true` | Allows attaching arbitrary metadata to active job runs for observability. |
| `FF_EXECUTION_TRACING` | `true` | Generates OpenTelemetry spans for detailed job execution lifecycle tracking. |
| `FF_COST_BUDGETS` | `true` | Enables tracking and enforcement of micro-USD cost limits for job runs. |
| `FF_AUDIT_LOG` | `true` | Records all configuration changes and job mutations in a persistent audit trail. |

## Job Organization

| Variable | Default | Description |
| :--- | :--- | :--- |
| `FF_ENVIRONMENTS` | `true` | Supports logical separation of jobs into environments like production and staging. |
| `FF_JOB_GROUPS` | `true` | Allows grouping related jobs for collective management and monitoring. |
| `FF_JOB_DEPENDENCIES` | `true` | Enables defining execution dependencies between different job types. |

## Queue and Scheduling

| Variable | Default | Description |
| :--- | :--- | :--- |
| `FF_ADAPTIVE_CONCURRENCY` | `true` | Dynamically adjusts worker concurrency based on system load and database performance. |
| `FF_EVENT_TRIGGERS` | `true` | Allows triggering job runs based on external events or database changes. |
| `FF_LISTEN_NOTIFY` | `true` | Uses PostgreSQL LISTEN/NOTIFY for low-latency job dispatching. |
| `FF_RATE_LIMIT_ENFORCEMENT` | `true` | Enforces global and project-level rate limits for job submissions. |
| `FF_PRIORITY_AGING` | `true` | Automatically increases the priority of long-waiting jobs in the queue. |
| `FF_QUERY_CACHE_WARMING` | `true` | Pre-populates internal caches to improve job lookup performance. |

## Webhooks

| Variable | Default | Description |
| :--- | :--- | :--- |
| `FF_WEBHOOK_CIRCUIT_BREAKER` | `true` | Protects the webhook dispatcher from failing endpoints. |
| `FF_WEBHOOK_SUBSCRIPTIONS` | `true` | Enables granular subscription to job lifecycle events via webhooks. |

## Opt-in

| Variable | Default | Description |
| :--- | :--- | :--- |
| `FF_CONCURRENCY_LIMITS` | `false` | Enforces strict limits on the number of concurrent runs per job definition. |
| `FF_PROJECT_QUOTAS` | `false` | Limits the total number of jobs and runs allowed within a project. |
| `FF_EXECUTION_WINDOWS` | `false` | Restricts job execution to specific time windows or maintenance periods. |
| `FF_QUEUE_PARTITIONING` | `false` | Enables logical partitioning of the job queue for multi-tenant isolation. |
| `FF_PROGRESS_STREAMING` | `false` | Supports real-time streaming of job progress updates via SSE. |
| `FF_CHECKPOINTS` | `false` | Allows jobs to save intermediate state for resumption after failure. |
| `FF_RUN_CONTINUATION` | `false` | Enables long-running jobs to spawn continuation runs for extended processing. |
| `FF_USAGE_TRACKING` | `false` | Collects detailed resource usage metrics for every job execution. |
| `FF_ERROR_CLASSIFICATION` | `false` | Automatically categorizes job failures based on error patterns and exit codes. |
| `FF_SECRET_INJECTION` | `false` | Securely injects encrypted secrets into the job execution environment. |
| `FF_RUN_REPLAY` | `false` | Allows re-executing completed or failed runs with the original input. |
| `FF_DRY_RUN` | `false` | Supports simulating job execution without performing side effects. |
| `FF_RUN_RETENTION` | `false` | Automatically purges old run data based on configured retention policies. |
| `FF_DEBUG_BUNDLE` | `false` | Generates a diagnostic bundle containing logs and state for failed runs. |
| `FF_BATCH_JOB_OPS` | `false` | Enables performing bulk operations on multiple jobs or runs simultaneously. |
| `FF_JOB_HEALTH_SCORING` | `false` | Calculates aggregate health scores based on success rates and latency. |
| `FF_ADAPTIVE_TIMEOUT` | `false` | Adjusts job timeouts dynamically based on historical execution data. |
| `FF_CONCURRENCY_ENFORCEMENT` | `false` | Strictly prevents exceeding configured concurrency limits at the worker level. |
| `FF_PROJECT_FAIR_QUEUE` | `false` | Ensures fair resource distribution among different projects in the queue. |
| `FF_WEBHOOK_DELIVERY_WORKER` | `false` | Uses a dedicated background worker pool for webhook delivery. |
| `FF_WEBHOOK_TIMESTAMP_SIGNATURES` | `false` | Includes cryptographic timestamps in webhook signatures for replay protection. |
| `FF_FALLBACK_ENDPOINT` | `false` | Supports defining fallback URLs for job dispatch when primary endpoints fail. |
| `FF_RUN_EVENTS_BUFFERED` | `false` | Buffers job lifecycle events to reduce database write pressure. |
| `FF_REDIS_REQUIRED` | `false` | Makes a Redis connection mandatory for application startup. |

## Dependencies

<Note>
  Some feature flags require additional environment variables to be configured.
</Note>

- `FF_SECRET_INJECTION` requires `SECRET_ENCRYPTION_KEY`
- `FF_ADAPTIVE_CONCURRENCY` uses `ADAPTIVE_CONCURRENCY_MIN/MAX`
- `FF_QUEUE_PARTITIONING` uses `WORKER_PARTITIONS/WORKER_PARTITION_WEIGHTS`
- `FF_WEBHOOK_DELIVERY_WORKER` enables background webhook polling
- `FF_REDIS_REQUIRED` makes Redis connection mandatory (fails startup if unavailable)
