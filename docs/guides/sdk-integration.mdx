Strait provides a set of SDK endpoints that allow your job code to communicate its progress, logs, and state back to the system.

## SDK Authentication

All SDK endpoints are located under `/sdk/v1/runs/{runID}/` and require a **JWT Run Token** for authentication.

- **Header**: `Authorization: Bearer <run_token>`
- **Token Source**: The `run_token` is provided in the response when a job is triggered.

## Available Endpoints

### Logging
`POST /log`
Send structured logs from your job.
- **Fields**: `message` (required), `level` (info, warn, error), `type`, `data` (JSON).

### Progress Updates
`POST /progress`
Report the completion percentage of a job.
- **Fields**: `percent` (0-100), `message` (required), `step`, `eta_seconds`.

### Annotations
`POST /annotate`
Add or update metadata annotations on the run.
- **Limit**: Max 50 annotations per run.

### Heartbeat
`POST /heartbeat`
Signal that the job is still alive and executing. This prevents Strait from marking the run as timed out or crashed.

### Checkpoints
`POST /checkpoint`
Save the current state of a resumable job.
- **Fields**: `state` (required JSON), `source`.

### Usage Tracking
`POST /usage`
Report AI model usage and costs.
- **Fields**: `provider`, `model`, `prompt_tokens`, `completion_tokens`, `cost_microusd`.

### Tool Calls
`POST /tool-call`
Record the invocation of an external tool or API.
- **Fields**: `tool_name`, `input`, `output`, `duration_ms`, `status`.

### Structured Outputs
`POST /output`
Store structured results or artifacts produced by the job.
- **Fields**: `output_key`, `value`, `schema` (optional JSON schema).

### Terminal States
- `POST /complete`: Mark the run as successfully completed. Optionally include a `result`.
- `POST /fail`: Mark the run as failed. Requires an `error` message.

### Wait for Event
`POST /wait-for-event`
Pause the current run and wait for an external event to arrive. The run transitions to `waiting` status.
- **Fields**: `event_key` (required, globally unique), `timeout_secs` (default: 3600), `notify_url` (optional webhook).
- **Behavior**: When the event arrives via `POST /v1/events/{eventKey}/send`, the run is re-queued with the event payload as checkpoint data and re-dispatched to your endpoint.

### Control Flow
- `POST /spawn`: Create a child job run.
- `POST /continue`: Create a continuation run (for long-running jobs that need to restart with fresh timeouts).

## SDK Client Libraries

Official SDK clients are available for TypeScript and Python in the `sdk/` directory. These provide typed wrappers around the REST API for event trigger operations.

### TypeScript

```typescript
import { EventsClient } from "@strait/sdk";

const events = new EventsClient("https://strait.dev", {
  Authorization: "Bearer strait_...",
});

// Pause a run and wait for an external event
const trigger = await events.waitForEvent("run-123", {
  eventKey: "approval:order-789",
  timeoutSecs: 7200,
});

// Send an event to resolve the trigger
await events.sendEvent({
  eventKey: "approval:order-789",
  payload: { approved: true },
});

// Get trigger status
const status = await events.getEventTrigger("approval:order-789");

// Cancel a waiting trigger
await events.cancelEventTrigger("approval:order-789");
```

### Python

```python
from strait import EventsClient, WaitForEventOptions, SendEventOptions

client = EventsClient(
    "https://strait.dev",
    {"Authorization": "Bearer strait_..."}
)

trigger = client.wait_for_event("run-123", WaitForEventOptions(
    event_key="approval:order-789",
    timeout_secs=7200,
))

resolved = client.send_event(SendEventOptions(
    event_key="approval:order-789",
    payload={"approved": True},
))

status = client.get_event_trigger("approval:order-789")
client.cancel_event_trigger("approval:order-789")
client.close()
```

## Endpoint Availability

All SDK endpoints documented on this page are always available.

## Example: Reporting Progress

```bash
curl -X POST https://strait.dev/sdk/v1/runs/{runID}/progress \
  -H "Authorization: Bearer $RUN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "percent": 45,
    "message": "Processing images...",
    "step": "image_optimization"
  }'
```

<Tip>
  Use the `heartbeat` endpoint regularly (e.g., every 30 seconds) in long-running jobs to ensure Strait knows the job is still active.
</Tip>
